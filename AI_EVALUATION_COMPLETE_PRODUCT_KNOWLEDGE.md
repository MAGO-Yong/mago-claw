### 2.5 参数详解

#### 温度 (Temperature)
- **范围**：0 - 2
- **默认值**：0.7
- **含义**：
  - 0：完全确定性，输出最可能的结果
  - 0.7：平衡创意性和确定性
  - 2：最大创意性，输出多样但可能不稳定
- **使用场景**：
  - 评分任务：推荐0.3-0.5（稳定）
  - 生成任务：推荐0.7-1.0（平衡）

#### Top P
- **范围**：0 - 1
- **默认值**：1
- **含义**：
  - 1：考虑所有token
  - 0.9：只考虑概率最高的90%的token
  - 越低越确定
- **使用场景**：
  - 推荐不修改，保持默认值1

#### Max Tokens
- **范围**：1 - 4096
- **默认值**：2048
- **含义**：单次调用的最大输出长度
- **使用场景**：
  - 评估场景：推荐512-1024（评分结果一般较短）
  - 生成场景：推荐2048+

#### 速率限制
- **含义**：每分钟最多调用多少次
- **使用场景**：
  - 防止API被过度调用
  - 根据订阅等级设置

#### Step 2a：评估数据 - 在线链路模式

**这是最常用的模式**

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 评估方式 | 单选 | ✅ | 单SPAN / 全链路 / 结果 |
| TraceId筛选源 | 单选 | ✅ | 日志 / Jaeger |
| SPAN筛选条件 | 键值对表格 | ✅ | 用于筛选要评估的链路 |
| 链路采样率 | 百分比 | ✅ | 0-100% |

**真实例子（sug粗筛任务）**：
```
评估方式: ○ 单SPAN (已选) ○ 全链路评估 ○ 结果评估
说明: 基于SPAN"sug"的输出进行评估

TraceId筛选源: ○ 日志 (已选) ○ Jaeger

SPAN筛选条件:
  ├─ name = sug
  └─ "+ 添加筛选器"

链路采样率: 7% (每1000条链路采样70条)
```

**字段详解**：

**评估方式选项**：
- **单SPAN**：
  - 选择一个特定的SPAN节点
  - 只评估该节点的输出
  - 用途：评估某个环节的质量
  - 例：name = sug 意思是评估name为"sug"的SPAN

- **全链路评估**：
  - 评估整个调用链路
  - 用途：评估端到端的性能
  
- **结果评估**：
  - 只评估最终结果
  - 用途：评估终态质量

**TraceId筛选源**：
- **日志**：从日志系统获取TraceId
- **Jaeger**：从Jaeger追踪系统获取TraceId

**SPAN筛选条件**：
- 用键值对定义
- 例：name = sug
- 支持多条件组合
- 操作符：=, >=, <=, >, <, !=

**链路采样率**：
- 范围：0-100%
- 说明：7%意味着每1000条链路中采样70条
- 用途：控制评估成本和覆盖率
- 实际例子：
  - 100%：所有链路都评估（最全面但成本高）
  - 7%：采样1/14的链路（成本低但覆盖度中等）
  - 1%：采样1/100的链路（最便宜但可能遗漏）

#### Step 2b：评估数据 - 离线数据集模式

**这是用于回放评估的模式**

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 目标数据集 | 下拉 | ✅ | 选择要评估的数据集 |
| 对象类型 | 下拉 | ✅ | 评估对象的类型 |
| HTTP服务配置 | 表格 | ✅ | 配置如何调用目标服务 |
| 请求头配置 | 动态表格 | ❌ | HTTP请求头 |
| 请求体配置 | 动态表格 | ❌ | HTTP请求体字段映射 |

**真实例子（某个数据集回放评估任务）**：
```
目标数据集: 测试回流数据集 (下拉选择)

对象类型: 自定义义模型 (下拉选择)

HTTP服务配置:
  ├─ Name*: XPoint
  ├─ Domain*: https://xray.devops.xiaohongshu.com
  ├─ Path*: /iagent/api/v1/chatbot/chat
  ├─ Method*: POST
  └─ IsStream: ☑ 是否流式传输

请求Header(动态表格):
  ├─ X-LangFuse-Source (String) → replay
  └─ X-LangFuse-Eval (String) → cml69xr500159ib7j22

请求Body(动态表格):
  └─ messages (Object) → [{"content":"$.input[0]"}]
```

**关键理解**：
- 这个模式会遍历数据集中的所有593条数据
- 对每条数据构建HTTP请求
- 调用目标服务获取评估结果
- 支持复杂的字段映射（用JsonPath表达式）

#### Step 3：评估器选择

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 评估器 | 下拉 | ✅ | 从已创建的评估器选择 |
| 输出字段映射 | 表格 | ✅ | 将评估结果映射到哪些字段 |

**真实例子（sug粗筛任务）**：
```
评估器*: sug粗筛 (下拉选择)

输出字段映射(表格):
  ├─ 行1: response_content → $ response_content
  └─ 行2: messages → $ messages[?(@.role == 'user')]
```

**字段映射说明**：
- **源字段**：评估器返回的字段名
- **目标字段**：要映射到的目标位置
- **支持JsonPath表达式**：用于复杂的字段提取

#### Step 4：数据回流（可选但强烈推荐）

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 是否启用回流 | 开关 | ❌ | 默认启用 |
| 目标数据集 | 下拉 | ✅ | 回流到哪个数据集 |
| 版本管理策略 | 单选 | ✅ | 新建版本 / 追加到现有 |
| 回流条件 | 表格 | ✅ | 满足条件才回流 |
| 字段映射 | 表格 | ✅ | 如何映射到数据集字段 |

**真实例子（sug粗筛任务）**：
```
是否启用回流: ✓ (已启用)

目标数据集: Sug粗筛数据集 (下拉选择)

版本管理策略: ○ 新建版本 (已选) ○ 追加到现有

回流条件(表格):
  ├─ 评估器: sug粗筛
  ├─ 条件表达式: score >= 2
  └─ "+ 添加条件"

字段映射(表格):
  ├─ messages ← $ data.input.messages[?(@.role == 'user')]
  └─ raw_input ← $ data.input.raw_input
```

**回流条件说明**：
- **条件类型**：score >= 2
- **含义**：只有评分大于等于2的链路才回流
- **作用**：自动过滤，只保留质量较好的数据
- **支持多条件**：可以用AND/OR组合多个条件

**版本管理策略说明**：
- **新建版本**：每次回流都创建新版本（推荐）
- **追加到现有**：追加到数据集的最新版本

**数据回流的意义**：
1. 自动发现新的edge case
2. 持续扩展测试数据集
3. 完成评估→学习→改进的循环
4. 版本控制自动记录历史

## 第八部分：关键技术细节

### 8.1 JsonPath表达式

**用途**：在字段映射中提取复杂的嵌套数据

**真实例子**：
```
$ response_content              # 获取最顶层的response_content字段

$ messages[?(@.role == 'user')] # 获取messages数组中role为'user'的所有项

$ data.input.messages[-1].content  # 获取data.input.messages数组的最后一个元素的content
```

**表达式说明**：
- `$` 代表根节点
- `.` 用于访问对象属性
- `[]` 用于访问数组
- `?(@.role == 'user')` 用于过滤数组

### 8.2 条件表达式

**用途**：在数据回流时设定条件

**真实例子**：
```
score >= 2      # 分数大于等于2

score >= 2 AND status = "ACTIVE"    # 多条件AND

score >= 2 OR confidence > 0.9       # 多条件OR
```

### 8.3 评估器与数据集的关系

```
评估器定义"怎么评"
  ↓
自动评估任务中选择评估器
  ↓
任务运行生成评估结果
  ↓
满足回流条件的结果保存到数据集
  ↓
该数据集可被下一个任务的"离线数据集"模式使用
  ↓
形成闭环
```
