# ai现状自述-大林 副本

## part 2：我推动团队 AI 实践的情况

**实践一：AI 平台孵化——XPilot（Agentic AIOps 平台）**

**1. 实践内容：**
2024 年主导发起「技术风险 + AI」创新方向，立项 XPilot，推动团队从传统 AIOps 工具体系向 Agentic 自治范式跃迁。分三个阶段推进：

- **第一阶段（7-8 月）**：确立"高频优先、信任渐进、技术可行"的产品策略，在数据洞察（Text2PQL、大盘智能解读、风险挖掘）和智能诊断（告警诊断、发布主动诊断、性能劣化诊断）两个核心方向率先落地，建立用户信任基础
- **第二阶段（9-10 月）**：基于用户分层判断，构建 AI 工作台 + 产品内嵌 + IM 端三端触达体系，让不同类型用户都能在各自工作流中获得 AI 能力支撑
- **第三阶段（11 月-次年 1 月）**：将平台能力 MCP 化，定义「技能」和「数字分身 + 定时任务」的产品形态，让业务和 SRE 可以自助孵化专属 AI 能力，实现从平台供给向生态化演进

**2. 这个实践是怎么发起的，我在其中扮演什么角色：**
方向完全由我主动判断发起，核心驱动来自对 AI Agent 发展趋势的提前预判——AI Agent 范式将重构稳定性工作的人机协作模式，平台价值将从"功能供给"转向"垂直领域最佳实践的 Skill 化积累"。我负责整个方向的战略判断、产品定义和阶段节奏把控。

这一判断在后续被行业主流方案相继印证：XPilot 定义的 AI 工作台、Skill 体系、数字分身 + 定时任务，分别对应了后来广受关注的 OpenClaw Agent 容器、Claude Code Skill 范式、OpenClaw 个人助手 + Cron 任务——XPilot 的落地比 Claude Code Skill 早约一个月，比 OpenClaw 相关功能早 2-3 个月。我们不是在跟随行业趋势，而是在垂直领域独立走到了同一个答案，并且更早。

**3. 目前的进展和可观测效果：**
XPilot 已形成完整产品矩阵：大盘解读效率提升 6 倍、性能劣化诊断效率提升 10 倍、智能客服承担 20% 工单量、90+ 用户建立个人数字分身、业务孵化 20+ 自定义技能。

**实践二：组织 AI 化——推动团队全角色 AI 工作方式转型**

**1. 实践内容：**

| 角色 | 实践内容 | 可用工具 | 目标效果 |
| ---- | ---- | ---- | ---- |
| 研发 | 全栈自闭环 | Cursor / Claude Code | 前端聚焦面向用户的产品功能自闭环，后端向数据层、引擎层、Skill 层等高复杂度工作深耕，各自在 AI 加持下发挥更大价值 |
| SRE | 能力放大与自闭环 | XPilot Agent + 技能平台 | 每个 SRE 拥有 AI 数字分身，将个人经验沉淀为可复用 Skill，实现稳定性建设从"个人依赖"到"组织能力"的跃迁 |
| PM | 产品验证模式升级 + AI 先行官 | Vibe Coding（Cursor 等） | 用可交互原型替代文档驱动，快速验证产品想法；亲身走完研发全链路，在 AI 编码、Agent 工程等实践中持续发掘新产品机会；同时作为团队 AI 先行官，持续跟踪前沿方向、过滤有价值信息、输出团队可对齐的方向判断，帮助团队在正确的时间押注正确的方向 |

**2. 这个实践是怎么发起的，我在其中扮演什么角色：**
基于对团队结构（研发、SRE、PM）和 AI 工具成熟度的综合判断，我主导推动各角色差异化的 AI 转型路径，而非统一要求。核心逻辑是让每个角色在 AI 加持下聚焦更高价值的工作，而不是简单地"用 AI 工具提效"。

**3. 目前的进展和可观测效果：**
研发 AI 编码持续推进中；SRE 数字分身和技能自闭环已落地，90+ 用户建立个人分身；PM Vibe Coding Q2 启动。

## part 3：我对团队工作的 AI 赋能判断

**1. 当前落地最大的阻力是什么：**
- **范式转型的认知门槛**：从"人用工具"到"人与 Agent 协作"是工作方式的根本性转变，用户信任需要通过可验证的结果逐步积累，不能一步到位
- **垂直领域经验的沉淀深度**：通用 AI 能力容易复制，真正的护城河是垂直领域最佳实践的 Skill 化积累，这需要持续投入
- **平台 Agent Native 化的转型成本**：打破现有面向人的产品形态，重新定义平台价值主张，需要团队在产品和技术上的思维同步升级

**2. 我计划何时推动：**
- **针对认知门槛**：Q2 推动 PM Vibe Coding，让产品侧亲身走完 AI 全链路，建立第一手认知，带动团队整体对 Agent 协作模式的理解
- **针对经验沉淀**：持续深化 SRE 与业务的共创机制，将更多垂直场景的排障经验固化为 Skill，扩大技能生态覆盖广度
- **针对转型成本**：Q3 重点推进平台 Agent Native 化，以核心能力 Skill 化为切入点，逐步重构平台价值交付方式
