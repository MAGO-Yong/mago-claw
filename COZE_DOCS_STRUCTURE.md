## 🎯 COZE文档的设计特点

### 1. 三栏布局（vs Langfuse的单栏）

**COZE**:
- 左：完整的导航树（可扩展/收缩的层级菜单）
- 中：主内容区域
- 右：目录快速跳转（Table of Contents）

**优势**:
- ✅ 左导航提供完整的文档地图
- ✅ 右目录快速定位页面内容
- ✅ 适合"大型文档库"（教程多、内容深）

### 2. 清晰的信息层次

**结构**:
```
# 标题（什么是评测）
  
## 定义段落
- 一句话定义
- 展开说明（包含key terms加粗）

## 离线评测
- 定义
- 使用场景
- 说明

## 在线评测
- 定义
- 使用场景
- 说明

## 功能模块（三个核心：评测集、评估器、实验）
- 评测集
  - 输入数据（定义）
  - 预期输出（定义）
  
- 评估器
  - 指标得分（定义）
  - 原因分析（定义）

- 实验
  - 评测集数据（定义）
  - 评测对象输出结果评分（定义）
  - 评估器打分（定义）

## 为什么选择
- 项目列表（每项都有描述）

## 快速开始
- 链接列表（四个快速入口）

## 计费
- 具体说明
```

### 3. 表现形式的特点

| 特点 | 体现 |
|------|------|
| **加粗关键词** | 在段落中加粗新概念（如"离线评测"、"人工校准评估"、"包括功能性"、"包括经济性"） |
| **列表而非段落** | "为什么选择"部分用●列表，每个要点有标题+描述 |
| **嵌套结构** | 评测集下有"输入数据"和"预期输出"的子项 |
| **右侧目录** | 快速导航到各部分（# 标题级别） |
| **快速入门卡片** | 四个入门路径，以按钮/链接形式展示 |
| **上下页导航** | 文档树中的前后页面链接 |

### 4. 内容策略

**定义优先**:
```
[concept]: [简短定义]
[详细说明，包含context]
[为什么需要]
[如何使用]
```

**例如**:
> **评测**是一种通过结构化评估框架，对评估对象进行全面质量监控和优化的过程...

> **离线评测**：在应用上线前，使用预设的评测集进行效果测试...

## 🔍 COZE vs Langfuse - 关键差异

| 维度 | COZE | Langfuse |
|------|------|---------|
| **导航** | 左边栏树形菜单 | 侧边栏菜单 |
| **右侧** | 目录快速跳转 | 无 |
| **定义位置** | 文章中段，加粗key terms | 概念页专门讲解 |
| **例子** | 问题→解决方案→快速开始 | 完整场景示例 |
| **对比表** | 列表（无表格） | 表格对比 |
| **可视化** | 极少 | 流程图、结构图 |
| **快速开始** | 链接列表 | 三步清单 |
| **文档规模** | 适合中等规模（100+页） | 适合小规模（20-50页） |

## 💡 关键洞察

### COZE的设计理念

1. **层级化导航**：充分利用左侧菜单建立"整体地图"
2. **快速定位**：右侧TOC让用户秒速找到内容
3. **词汇加粗**：通过排版强调新概念（而非冗长解释）
4. **列表组织**：大量使用bullet list而非段落
5. **概念嵌套**：子概念在列表中进行分层展开

### 适应场景

- ✅ 文档体系大（多个模块、子主题多）
- ✅ 需要频繁跳转（用户需要多次导航）
- ✅ 概念众多（需要树形组织）
- ✅ 希望"快速浏览"而非"细致理解"

### 不适应场景

- ❌ 文档体系小（页数少，导航简单）
- ❌ 需要"完整故事"（缺少端到端示例）
- ❌ 强调概念关系（需要多个图表）

## 📊 对你项目的启示

### 学习点

1. **导航设计**：AI应用评估有5个模块，适合做成树形导航
   ```
   AI应用评估
   ├─ 什么是评估
   ├─ 核心概念
   │  ├─ 两种模式
   │  ├─ 两种评估器
   │  └─ 工作流程
   ├─ 快速开始
   ├─ 详细指南
   │  ├─ LLM配置
   │  ├─ 评估器
   │  ├─ 评估任务
   │  └─ 结果分析
   └─ 高级功能
   ```

2. **词汇加粗**：在段落中突出新概念（在线链路模式、离线数据模式等）

3. **列表组织**：三个核心概念可用bullet list而非段落

4. **快速开始**：不是三步清单，而是四个链接卡片
