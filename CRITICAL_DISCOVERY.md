# 🚀 关键发现：数据集模式的自动评估

**发现时间**：2026-03-10 11:59 GMT+8  
**重要性**：⭐⭐⭐⭐⭐ **非常关键！**  
**影响**：完全改变了第2步的配置方式

---

## 📌 核心发现

**AI应用评估系统支持两种完全不同的评估模式：**

### 模式A：在线链路模式（之前探索的）
```
生产环境实时数据
    ↓
TraceId筛选 → SPAN筛选 → 采样
    ↓
评估
    ↓
结果回流
```

### 模式B：数据集模式（新发现！）🎯
```
预先准备的数据集
    ↓
全量无筛选
    ↓
HTTP服务调用（完整的API配置）
    ↓
评估
    ↓
结果回流
```

---

## 🔍 第2步的根本区别

### 在线链路模式 - Step 2

```
评估方式：单SPAN | 全链路 | 结果评估
TraceId筛选：日志 | Jaeger
SPAN筛选：name = sug
采样率：7%
```

### 数据集模式 - Step 2 ⭐⭐⭐

```
目标数据集*：测试回流数据集（下拉选择）
是否开启回流：✓启用/☐关闭

对象类型*：自定义义模型

HTTP配置信息
├─ Name*：XPoint
├─ Domain*：https://xray.devops.xiaohongshu.com
├─ Path*：/iagent/api/v1/chatbot/chat
├─ Method*：POST
├─ IsStream：☑ 是否流式传输

请求Header配置（动态表）
├─ X-LangFuse-Source (String) → replay
├─ X-LangFuse-Eval (String) → cml69xr500159ib7j22
└─ "+ 添加Header"

请求Body配置（动态表）
├─ messages (Object) → [{"content":"$.input[0]"}]
└─ "+ 添加Body字段"
```

## 💡 关键洞察

### 1. 完全的HTTP API集成

数据集模式允许配置：
- ✅ 完整的HTTP请求头（支持自定义）
- ✅ 完整的HTTP请求体（支持JsonPath映射）
- ✅ 流式传输选项
- ✅ 自定义的Domain/Path/Method

这意味着可以**调用任何HTTP服务进行评估**！

### 2. 灵活的字段映射

```
请求Body示例：
messages: [{"content":"$.input[0]"}]

说明：
- 从数据集的$.input字段
- 提取第一个元素[0]
- 作为HTTP请求的messages字段

支持完整的JsonPath表达式！
```

### 3. 回放能力

请求Header中的标记：
```
X-LangFuse-Source: replay
说明这是一个"回放"请求，而不是实时请求
允许服务区分和记录不同的请求类型
```

### 4. 评估ID追踪

```
X-LangFuse-Eval: cml69xr500159ib7j22
用于追踪这个评估任务的唯一ID
方便服务端记录日志和追踪
```

## 🎯 应用场景

### 场景1：离线A/B测试

```
数据集：历史真实用户对话 (593条)

对比方案A：
- 调用模型A的HTTP服务
- 评估其在这些真实对话上的性能

对比方案B：
- 调用模型B的HTTP服务
- 评估其在同样数据上的性能

结果：可以客观对比哪个模型更好
```

### 场景2：服务升级验证

```
升级前：
- 在数据集上运行旧服务
- 记录评估结果

升级后：
- 在同样数据集上运行新服务
- 比对评估结果
- 确保没有性能下降
```

### 场景3：多模型对比

```
数据集：通用评估数据

任务1：调用服务A评估
任务2：调用服务B评估
任务3：调用服务C评估

对比三个服务在相同数据上的性能
```

### 场景4：精细化评估

```
生产环境虽然有实时数据，但：
- 无法重复执行（链路已过期）
- 无法精确控制测试条件

使用数据集模式：
- 可以反复执行
- 精确控制输入
- 稳定的评估环境
```

## 🔄 完整的两种模式对比

| 维度 | 在线链路模式 | 数据集模式 |
|------|------------|---------|
| **数据源** | 实时生产环境 | 预先准备的数据集 |
| **采样** | 需要配置采样率 | 无采样（全量） |
| **筛选** | 需要TraceId + SPAN筛选 | 无筛选 |
| **评估方式** | 支持3种 | N/A（由服务决定） |
| **HTTP配置** | 无需 | ✓完整HTTP配置 |
| **重复性** | 无法重复（链路一次性） | ✓完全可重复 |
| **对比性** | 难以A/B对比 | ✓易于对比 |
| **场景** | 实时监控 | 离线评估、对比、回放 |
| **成本** | 随流量增长 | 固定成本 |
| **准确性** | 依赖生产流量特征 | 可控的测试数据 |

## 🛠️ 如何使用数据集模式

### Step 1：准备数据集
```
从之前的评估中回流得到的数据集
或者手动导入的测试数据
```

### Step 2：创建自动评估任务（数据集模式）
```
1. 在创建任务时选择"数据集数据源"
2. 第2步填写：
   - 目标数据集
   - HTTP服务配置
   - 请求头/请求体映射
3. 后续步骤与在线链路模式相同
```

### Step 3：定期执行
```
设置执行周期（每天00:30等）
系统自动调用HTTP服务
评估数据集中的全部数据
```

### Step 4：查看结果
```
在评估实验中查看结果
对比不同时期的评估数据
追踪服务性能变化
```
