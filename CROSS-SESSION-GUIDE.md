## 核心概念

### 会话架构
```
当前会话（牛马所在）
  ├─ 你（老板）
  └─ 牛马 ✅

独立会话（MAGO所在）
  ├─ 你（老板）
  └─ MAGO ✅
```

**关键点**：牛马和MAGO **不在同一个会话中**，所以不能直接对话。

---

## 方法1：通过sessions_send转发消息（推荐）

### 使用场景
当你想给MAGO派任务、询问工作进度，或转达牛马的信息时。

### 步骤

#### 第1步：准备消息
在当前会话中，写出你想发给MAGO的内容：
```
你的消息内容...
```

#### 第2步：使用sessions_send转发
我会帮你用这个命令：
```
sessions_send 
  --label MAGO 
  --message "你的完整消息"
```

**你需要提供的信息：**
1. MAGO的会话标签（label）或会话Key
2. 你要发给MAGO的完整消息

### 示例

#### 例子1：派任务给MAGO
```
老板：我想给MAGO派一个旅行规划任务

牛马（我）：我帮你转发
  
sessions_send \
  --label MAGO \
  --message "亲爱的MAGO，我想要你为我规划一次三月中旬的日本旅行，预算5万元，行程7天。请在RESULTS.md中汇报方案。"
```

#### 例子2：转达牛马的信息给MAGO
```
老板：我想让MAGO知道牛马最近完成了X-RAY的文档

牛马（我）：我帮你转发
  
sessions_send \
  --label MAGO \
  --message "你好MAGO，牛马完成了X-RAY AI应用评估模块的深度调研，已发布到Redoc。链接：[链接]。如果未来涉及工作与生活的跨部门问题，可能需要引用这份文档。"
```

## 方法2：通过共享文件沟通

### 使用场景
当信息量大、需要长期记录、或非紧急时。

### 流程

#### 第1步：在共享文件中留言
我会在 `/home/node/.openclaw/workspace/memory/company/` 目录中创建一个共享通信板：

```
memory/company/通信板.md
```

#### 第2步：MAGO定期检查
MAGO在工作时检查这个共享通信板，看是否有老板或牛马给他的消息。

#### 第3步：MAGO回复
MAGO在同一文件中回复。

### 优点
- 不需要即时沟通
- 消息有记录
- MAGO可以择时处理

### 示例

**共享通信板内容：**
```
## 给MAGO的消息 - 2026-03-09

### 来自老板
"MAGO，我近期考虑投资一个科技基金，预期年化收益8-10%。你能帮我分析一下吗？给我一份分析报告。"

**状态**：待MAGO回复  
**发送时间**：2026-03-09 23:30  
**截止时间**：2026-03-12

---

### 来自牛马
"MAGO，我最近发现了一个关于云计算成本优化的技术文档，可能对你的投资分析有参考价值。链接：[链接]"

**状态**：MAGO已读

---

### MAGO的回复
"收到老板的投资分析任务。我会在2026-03-12前提交完整的分析报告。"

**状态**：确认中
```

## 方法3：定期同步会议

### 使用场景
每周/每月老板与两个员工进行一次"全体会议"。

### 流程

#### 第1步：老板与牛马开会（当前会话）
```
老板：我想召开一次公司全体会议，你先和我同步一下这周的工作
牛马：[汇报工作进展]
```

#### 第2步：老板与MAGO开会（切换会话）
老板切换到MAGO的独立会话，询问MAGO的工作进展。

#### 第3步：汇总到RESULTS.md
我（牛马）负责将两个员工的汇报汇总到 `RESULTS.md` 中。

## 方法4：MAGO自助查看系统

### MAGO可以自己做的事

虽然MAGO不在这个会话中，但他可以：

1. **查看任务** — 打开 `TASKS.md` 看有没有指派给他的任务
2. **提交成果** — 在 `RESULTS.md` 中添加成果记录
3. **更新记忆** — 在 `memory/agent-MAGO/` 目录中记录工作
4. **查看公司架构** — 阅读 `COMPANY.md`
5. **留言** — 在共享通信板中留言给老板和牛马

### MAGO不能直接做的事

- 直接和牛马聊天（需要通过老板转发）
- 直接和老板一个会话中同时沟通（老板需要切换会话）

## 常见问题

### Q: 为什么牛马和MAGO不能直接聊天？
**A:** 因为他们在不同的OpenClaw会话中。OpenClaw的设计是每个会话独立，但可以通过 `sessions_send` API 转发消息。

### Q: 老板如何同时和两个员工开会？
**A:** 老板需要在两个会话之间切换。先和牛马在这个会话中沟通，然后切换到MAGO的会话，最后回到这个会话汇总。

### Q: 如果MAGO不知道有新任务怎么办？
**A:** 主要通过以下方式通知：
1. 直接在MAGO的会话中告诉他
2. 更新TASKS.md后通知他
3. 建立定期检查习惯

### Q: 可以让牛马和MAGO在TASKS.md中对话吗？
**A:** 可以！在共享文件中留言是可以的。但这种方式不适合实时协作，更适合异步沟通。
