## 第一部分：什么是评估

### 1.1 核心定义

**AI应用评估系统用于自动化地衡量AI服务的输出质量。**

系统的核心功能：
- **自动采样**：从生产环境或数据集中获取样本
- **自动评估**：使用预定义的评估器对样本进行评分
- **结果存储**：保存评估结果和历史记录
- **趋势分析**：追踪质量变化趋势

### 1.2 为什么需要

| 问题 | 传统做法 | 使用评估系统 |
|------|--------|-----------|
| 如何知道AI质量好不好？ | 人工审核（昂贵、缓慢） | 自动评估（持续、及时） |
| 新版本会不会变差？ | 上线后发现问题 | 上线前验证 |
| 长期质量如何变化？ | 无法追踪 | 完整的数据记录 |
| 如何对比两个方案？ | 主观判断 | 客观的评分对比 |

### 1.3 核心价值

- 📊 **可见性**：将无形的AI质量转变为可衡量的数据
- 🔍 **及时性**：实时发现质量问题，避免累积
- 🎯 **科学性**：基于数据而非主观判断的决策
- 💰 **经济性**：自动化评估比人工审核成本低50-90%

## 第二部分：名词解释

### 2.1 链路 (Trace)

**定义**：一条完整的AI处理过程，从用户输入到系统输出。

```
用户输入
  ↓
[AI处理（可能涉及多个步骤）]
  ↓
系统输出

↑ 整个过程称为一条"链路"
```

**在评估中的作用**：我们评估的基本单位就是链路。

**示例**：
```json
{
  "trace_id": "trace_12345",
  "timestamp": "2026-03-10T12:00:00Z",
  "input": "今天天气怎么样",
  "output": "根据最新天气数据...",
  "metadata": {
    "user_id": "user_123",
    "service": "weather_bot"
  }
}
```

### 2.2 SPAN（跨度）

**定义**：链路中的某个具体环节或步骤。

```
[输入处理] → [API调用] → [数据处理] → [输出格式化]
   SPAN1       SPAN2       SPAN3         SPAN4
```

**在评估中的作用**：允许精细化评估。可以只评估某个环节而非整条链路。

**示例场景**：
- 只评估"API调用"这个步骤是否成功
- 只评估"输出格式化"的效果

### 2.3 评估器 (Evaluator)

**定义**：用于判断AI输出质量的规则或模型。

**两种类型**：

#### 2.3.1 LLM评估器
- **原理**：使用另一个LLM来评价AI的输出
- **配置方式**：编写Prompt（评估标准）
- **成本**：相对便宜
- **精度**：中等

#### 2.3.2 HTTP代码评估器  
- **原理**：调用外部HTTP服务进行评估
- **配置方式**：配置HTTP服务地址和参数映射
- **成本**：取决于服务
- **精度**：高（完全可控）

### 2.4 评估任务 (Evaluation Task)

**定义**：一个"评估计划"，定义了如何、何时、用什么来评估。

**包含内容**：
- 评估对象（哪些链路）
- 评估方式（在线/离线）
- 评估器选择（用哪个评估器）
- 执行频率（多久运行一次）

### 2.5 评估实验 (Experiment)

**定义**：一次评估任务的执行结果。

**包含内容**：
- 执行时间
- 评估链路数
- 平均评分
- 分数分布
- 详细的链路评估数据

## 第三部分：使用场景

### 3.1 实时质量监控

**场景**：持续监控生产环境中AI的质量，及时发现问题。

**配置**：
- 模式：在线链路模式
- 频率：每1小时或每天
- 评估器：快速评估器（LLM）

**输出**：质量趋势图表，异常告警

### 3.2 AB测试

**场景**：对比两个版本的AI服务，判断哪个更好。

**配置**：
- 模式：离线数据模式
- 数据：相同的测试数据集
- 评估器：相同的评估标准
- 对比：新版本 vs 旧版本

**输出**：可比较的评分结果，帮助决策

### 3.3 升级验证

**场景**：升级前进行质量基线测量，升级后验证是否有性能下降。

**流程**：
```
升级前评估 (baseline: 3.7分)
    ↓
执行升级
    ↓
升级后评估 (current: 3.8分)
    ↓
判断：新版本更好 ✓ 可以保留
```

### 3.4 性能基准

**场景**：建立长期性能基准，追踪质量演进。

**配置**：
- 定期评估（每周/每月）
- 使用相同的基准数据集
- 记录历史数据

**输出**：质量趋势曲线，识别改进机会

## 第四部分：核心概念

### 4.1 两种评估模式对比

| 维度 | 在线链路模式 | 离线数据模式 |
|------|------------|-----------|
| **数据源** | 实时生产数据 | 历史数据集 |
| **特点** | 最新、真实 | 可重复、可控 |
| **评估方式** | 支持单SPAN/全链路/结果 | 完整链路 |
| **采样方式** | 可配置采样率 | 全量或自定义 |
| **适用场景** | 实时监控 | AB测试、验证 |
| **成本** | 随流量增加 | 固定 |

## 第五部分：快速开始

### 5.1 前置准备

1. **了解现状**：确认你要评估的AI服务
2. **准备数据**：明确评估标准
3. **选择模式**：在线监控 或 离线对比

### 5.2 3步创建第一个评估

**Step 1：创建评估器（5分钟）**

进入系统 → 选择评估器 → 新建 → 选择类型

对于快速试验，推荐使用LLM评估器：

```
名称：基础质量评分
选择LLM：[选择可用的LLM]
Prompt：
评估以下AI的输出质量，评分1-5分：
- 1分：完全错误
- 5分：完美答案

问题：{question}
回答：{answer}

请返回JSON: {"score": 数字, "reason": "理由"}
```

**Step 2：创建评估任务（10分钟）**

评估 → 新建任务 → 配置

```
任务名称：我的第一个评估
数据来源：在线链路
执行周期：每天 00:30
评估器：基础质量评分（从Step 1选择）
```

**Step 3：查看结果（等待执行）**

等待自动执行 → 实验 → 查看评分
