## 五、从问答内容提炼的核心场景需求

从用户自由描述中提炼出以下高频、高价值场景：

### 🔴 P0 最高优先级场景

**1. 日志智能查询与分析（36人选，多人问答描述）**
- 根据关键词/traceId/uid+时间范围/接口名，快速查询日志
- AI 自动聚合分析错误日志（如哪类 error 突增），不用人工看
- 支持日志统计（类 SQL select group by），解决 CAT 打点维度不足的问题
- 代码 debug 时让 AI 直接读 xray 日志，实现开发测试闭环
- 典型引用：*"目前AI在开发上已经很强，现在卡点是每次测试需要去读xray日志，如果能让AI直接读xray日志，日常简单需求就能完成开发测试闭环"*

**2. 链路追踪与异常归因（24人查询 + 22人诊断）**
- 给定 traceId，自动拉取完整链路，展示报错信息、用户访问路径
- 从链路中识别上下游瓶颈（如 cpu 100% 导致 p99 升高）
- AI 自动聚类异常、定位根因代码，关联变更记录
- 典型引用：*"遇到报警 SQL 成功率低，xray查异常堆栈，同时关联 ones/apollo 查变更判断 - 这一步可以通过 skill 自动化"*

**3. 异常诊断智能化（26人选）**
- 告警触发后，AI 自动诊断原因（异常聚类 + 变更关联）
- 支持服务异常和性能问题的全流程自助分析（包括内存分析、pod 故障巡检等）
- 典型引用：*"堆内存分析，感觉可以全流程自助分析一下，包括关联近期业务变更、内存变化率全面分析"*

### 🟠 P1 高优先级场景

**4. 大盘/图表自动创建（17人选）**
- 用自然语言描述，AI 生成大盘图表（指定服务/指标/时间范围）
- 批量生成压测/灰度观测看板，支持对话式调整
- 将 xray 数据一键导入 RedBI 看板，避免二次配置
- 典型引用：*"根据我提供的指标名称和tag信息，直接帮我生成大盘和图表"*

**5. 告警规则自动化管理（15人选创建 + 11人选查询）**
- 用自然语言创建/修改告警规则（仿照已有规则、自动推荐阈值）
- 查询服务告警覆盖情况、验证告警有效性（SRE 巡检场景）
- 支持按个人静默（非全局），减少误扰
- 典型引用：*"仿照 url://xxx 规则写一个 for xx 的报警规则，阈值设为 yy/根据历史自动推荐阈值"*

**6. 指标查询与性能分析（26人选指标查询 + 18人选性能诊断）**
- 查询 pod/机器指标，定位异常 pod
- 服务发布后的性能对比观测（canary vs 老 pod，多集群对比）
- 典型引用：*"发版时观察特定日志量在 canary 和其他老 pod 是否有变化，每次发版要一个集群一个集群查，希望用自然语言让 Agent 自己查自己对比"*

### 🟡 P2 中优先级场景

**7. Agent/成本分析（12+16人选）**
- GPU 利用率、计算成本分析
- Agent 任务状态查询（larc 平台任务失败原因诊断）
- 典型引用：*"larc 平台自助诊断任务失败原因"*

**8. 日志定时下载/报表化（多人问答提及）**
- 定时统计日志导入 OLAP 引擎/RedBI 生成报表
- 每周固定字段日志统计
- 典型引用：*"希望可以将结果直接导入在线doc和定时统计导入olap引擎用于统计报表"*

## 六、综合结论与 SKILL 孵化优先级建议

### 第一梯队（立即孵化，需求最广泛）

| SKILL | 核心价值 | 目标用户 |
|-------|---------|---------|
| **日志查询 SKILL** | 支持自然语言查日志、关键词/traceId/uid 多维检索、AI 聚合分析错误 | 全体研发（78% 需求） |
| **异常诊断 SKILL** | 告警触发后 AI 自动分析：聚类异常 → 定位根因 → 关联变更 | 研发/SRE（57%） |
| **链路查询+链路诊断 SKILL** | traceId 输入 → 全链路展开 → 上下游瓶颈自动识别 | 后端/前端开发（52%/48%） |

### 第二梯队（重要，尽快跟进）

| SKILL | 核心价值 | 目标用户 |
|-------|---------|---------|
| **指标查询+性能诊断 SKILL** | 自然语言查指标、pod/机器负载、多集群对比分析 | 研发/SRE（57%/39%） |
| **大盘图表创建 SKILL** | 对话式建大盘、批量生成观测看板、一键导 RedBI | 研发/技术负责人（37%） |
| **告警规则管理 SKILL** | 创建/修改/查询/静默告警，支持自动推荐阈值和有效性巡检 | SRE/研发（33%） |

### 第三梯队（有需求，按资源排期）

| SKILL | 核心价值 | 目标用户 |
|-------|---------|---------|
| **成本分析 SKILL** | GPU/算力成本查询与分析 | AI平台/算法（35%） |
| **Agent状态分析 SKILL** | Agent 任务状态、失败原因诊断 | 算法/技术负责人（26%） |
| **大盘诊断 SKILL** | 大盘异常自动识别与告警 | 全体（22%） |
| **日志定时报表 SKILL** | 定时日志导出 + 自动写入 RedBI/OLAP | 有统计报表需求的研发 |
