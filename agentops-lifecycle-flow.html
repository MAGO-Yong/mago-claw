AgentDev × AgentOps 全生命周期联动

ALLIN/Darwin（创建侧）× 发布平台/XRay（治理侧）

1
创建期 · 注册即治理
Agent 创建的同时，AgentOps 完成注册、埋点配置、SLO 定义
Phase 1
AgentDev（ALLIN）
创建 Workflow
  • 定义 Agent 节点结构（DAG）
  • 选择开发框架（ARKAI / ALLIN / Darwin）
  • 配置工具列表（Tools）
  • 选择模型（模型名 + 版本）
触发注册事件
  • 发送 workflow.created 事件
  • 携带 agent_id、owner、intent、工具列表
AgentOps（发布平台 + XRay）
自动注册 Agent 元数据
  • 写入 Agent 元数据中心
  • 记录 agent_id、owner、intent、capability_boundary
  • 初始化 health 状态
配置观测埋点
  • 下发 AgentOps SDK 初始化参数
  • 指定 XRay App ID（上报目标）
  • 设置采样率：开发期 100%，生产期 10%
  • 标注关键节点必须打 Span
引导定义 SLO
  • task_success_rate 目标值
  • hallucination_rate 上限
  • p99_latency 阈值
  • 写入元数据中心，作为后续评估基准
2
开发期 · 基线先行
开发期 Trace 即上报，评估基线在上线前建立
Phase 1
AgentDev（ALLIN）
迭代 Workflow
  • 本地 / 测试环境调试
  • 修改 Prompt、调整节点逻辑
  • 工具参数联调
触发更新事件
  • 发送 workflow.updated 事件
  • 携带变更 diff（修改了哪个节点）
AgentOps（XRay + 评估）
开发态 Trace 实时上报
  • 创建即采集，不等上线
  • 开发期与生产期 Trace 同构，天然可对比
  • Tool 调用链、RAG 命中情况全部可查
建立评估基线
  • 用测试数据集跑离线 Eval
  • 记录当前版本的质量分（基线快照）
  • 更新元数据：baseline_eval_score
  • 上线后对比用，判断是变好还是变差
语义快照更新
  • 接收 workflow.updated 事件
  • 更新元数据中 prompt_hash、flow_diff
  • 触发基线重建（如变更幅度大）
3
发布期 · 评估门禁
评估结果作为硬性发布卡点，不达标不放行
Phase 2
AgentDev（ALLIN）
发起发布申请
  • 调发布平台接口，创建发布单
  • 携带 agent_meta 字段（AI 语义包）
agent_meta: {
  workflow_version: "v2.3.1",
  prompt_hash: "a3f9cd...",
  model: "gpt-4o",
  tools: ["search", "db_query"],
  flow_diff: { 新增节点 B }
}
AgentOps（发布平台）
写入变更语义记录
  • 发布单绑定 agent_meta（AI 语义包）
  • 记录：改了什么 Prompt、换了什么模型、增减了哪些工具
  • 更新元数据中心：current_version、change_diff
触发 Eval Gate（评估门禁）
  • 用历史数据集自动回放新版本
  • 评估指标对比基线
  • task_success_rate ≥ 85% → 放行
  • task_success_rate < 85% → 驳回 + 附评估报告
灰度流量管理
  • 默认灰度 1% 流量先上
  • 异常率监控，正常后扩量
  • 高危工具（如 exec_shell）强制人工审批
↕ 发布期双向事件
→
deploy.requested
ALLIN 发起 → AgentOps 触发 Eval Gate
←
eval.gate.passed / failed
AgentOps → ALLIN 回传评估结果，passed 放行 / failed 驳回
4
运行期 · 持续治理
持续评估采样，质量下滑自动关联变更并回流 AgentDev
Phase 2 → 3
AgentDev（ALLIN）
接收质量反馈
  • 查看 AgentOps 推送的质量告警
  • 查看异常归因（关联到哪次变更）
  • 响应回滚建议（人工审批后执行）
问题 Workflow 版本标记
  • AgentOps 回写：该版本 ⚠ 线上质量异常
  • 开发者在 ALLIN 侧可见质量状态
AgentOps（XRay + 评估）
持续 Eval 采样
  • 生产流量实时打分（LLM-as-Judge）
  • 质量指标持续对比 SLO 基线
  • 发现漂移：task_success_rate 91% → 76%
异常自动关联变更
  • 质量下滑 → 检索最近变更记录
  • 定位：下滑发生在 v2.3.1 发布后 30min
  • AI Trace × 基础设施异常联合分析
分级处置
  • 轻：在 ALLIN 打标 ⚠ 质量异常
  • 中：推送回滚建议单，等待人工审批
  • 重：P0 自动触发回滚（需提前授权）
↕ 运行期双向事件
→
workflow.deprecated
ALLIN 下线 → AgentOps 归档元数据
←
quality.degraded
AgentOps → ALLIN 质量下滑告警 + 关联变更版本
←
rollback.suggested
AgentOps → ALLIN 推送回滚建议（人工审批）
←
slo.violated
AgentOps → ALLIN SLO 违规告警
5
退役期 · 沉淀知识
历史数据归档，沉淀为下一个 Agent 的参考案例
Phase 3
AgentDev（ALLIN）
下线 Workflow
  • 发送 workflow.deprecated 事件
  • 标记该 Agent 生命周期结束
AgentOps
元数据归档
  • Profile 标记为 deprecated
  • 保留完整历史：版本记录、评估分、异常记录
沉淀为参考案例
  • 哪些 Prompt 版本效果好
  • 哪些变更类型容易引发质量下滑
  • 供下一个同类 Agent 创建时参考

传感器（持续评估） + 执行器（发布卡点 / 护栏 / 回滚） = 负反馈闭环
AgentDev 负责"定义 Agent 是什么"，AgentOps 负责"治理 Agent 怎么跑"——两平台共同维护 Agent 全生命周期状态
