核心能力 · 重点
在线链路评估：配置一次，持续自动运行

AI 应用接入 X-RAY 后，基于真实用户请求自动持续评估，不需要人工整理样本，也不需要人工盯着看。

🔧 怎么运行

1

确认链路已接入 X-RAY

trace、SPAN、输入输出等链路数据在 X-RAY 可见，这是在线评估的数据来源

2

创建评估器，定义好坏标准

用 LLM 评估器写 Prompt（相关性？正确性？幻觉？），或接入已有 checker 服务——只需定义一次

3

配置自动评估任务

选"在线链路数据"，设好筛选条件、采样率（控制 token 成本）、执行周期，设完之后不用管

4

自动产出报告 + 回流 bad case

每次任务运行后自动生成实验报告；低分样本按规则自动写进数据集，持续沉淀

📡 你可以持续监控

📈

整体效果趋势

得分分布（P50/P90/P99），版本上线后有没有出现整体波动

🎯

分场景精细评估

通过 Trace 筛选，只评你关心的那个链路，不被无关流量干扰

🔍

低分样本精准定位

筛出低分 bad case，查看评估理由，一键跳转完整链路追溯根因

⚠️

接入已有 checker 服务

幻觉检测、工具调用规则校验——业务已有的判分服务直接对接进来统一管理

在线评估粒度，三选一：
最常用
🔬

单 SPAN 评估

只评某一个具体 LLM 调用环节，如某次 RAG 检索、工具调用的输出质量。LLM / HTTP 评估器均可。

🔗

全链路评估

把完整 trace + 所有 observations 一起送给评估器，适合评整条 Agent 链路的综合效果。仅支持 HTTP 评估器。

🎯

结果评估

只关心最终给用户的 input / output，不感知中间过程，快速判断最终结果对不对。LLM / HTTP 均可。

产品结构
五个模块，一条完整的评估闭环

不是五个孤立工具，而是同一条评估链路中的不同环节。

⚙️
LLM 配置

给 LLM 评估器提供底层模型。评估效果和 token 成本由这里决定。支持配置温度（建议 0~0.3）、QPM，创建后先测连通性。

⚖️
评估器

定义"裁判标准"——评什么维度、怎么打分、给出什么理由。支持 LLM 和 HTTP 两类，强制输出 score + reason。

🗄️
数据集

评估样本库。线上 bad case 自动回流沉淀，或手动整理样本集。最多 1 万条，支持 JSON / 字符串等多种字段类型。

🤖
自动评估

整套产品的执行中心。配置一次，持续自动运行。串联数据来源、评估器、调度周期和回流策略，内置采样率控制。

📊
评估实验

每次任务运行后的结果报告。实验分析看整体分布（P50/P90/P99）；实验明细找低分样本，可跳转完整链路追溯根因。

快速开始
4 步建立第一条在线监控任务

适用于 AI 应用已接入 X-RAY、希望马上开始持续监控线上效果的场景。

1

配置底层模型

在「LLM 配置」中添加可用模型，测试连通性通过后备用

2

创建 LLM 评估器

写好 Prompt，定义评分标准，配置字段映射（从 SPAN 取哪些字段）

3

建自动评估任务

选"在线链路数据"，配 Trace 筛选、采样率（建议先 5%）、按天执行

4

触发 + 验证报告

先手动跑一次，抽查 reason 是否符合预期，再开启自动调度
