# 自动评估

## 1. 自动评估任务是什么

自动评估任务是整套产品的**执行中心**。前面的数据集、评估器和 LLM 配置，更多是在准备样本和规则；到了自动评估任务这一步，系统才真正把"评什么、用什么评、什么时候跑、跑完之后怎么处理结果"组织成一条可以持续执行的流程。

自动评估任务不是单纯的调度器，而是把**数据来源、评估方式、评估器、执行周期和回流策略**串在一起的核心配置单元。

三个容易混淆的概念，先区分清楚：

| 概念 | 含义 |
|------|------|
| **在线 / 离线** | 数据来源——链路数据来自线上实时流量，还是离线整理好的数据集 |
| **单SPAN / 全链路 / 结果评估** | 评估方式——评估粒度和输入范围（仅在线链路时有此选择）。注意：**选择全链路评估时，后续必须使用 HTTP 评估器**（LLM 评估器不支持全链路模式） |
| **评估器** | 判分规则——具体用什么规则打分 |

---

## 2. 开始前：先想清楚三件事

创建自动评估任务前，建议先回答清楚：

1. **数据来源**：这次评估的是线上链路，还是离线数据集？
2. **评估器**：准备挂哪些评估器？LLM 评估器对应的 LLM 配置是否可用？
3. **结果处理**：跑完之后，是否需要把特定分数段的样本回流沉淀到数据集？

这三个问题没想清楚直接进入配置流程，通常会在第二步和第三步之间反复跳转。

> **前置条件**：
> 1. 至少一个可用评估器。如果使用 LLM 评估器，还需先确认对应的 LLM 配置可连通。→ 参考：[评估器](https://docs.xiaohongshu.com/doc/0bced567cecfcacf14730e6c3490fcbf) / [LLM 配置](https://docs.xiaohongshu.com/doc/fe0671ef5a9436eb151f9a115cabbee4)
> 2. 明确数据来源：
>    - **离线数据集**：需提前准备好数据集，字段结构可以和评估器变量对齐。→ 参考：[数据集](https://docs.xiaohongshu.com/doc/26203de6abac2f78a8a1e0fa39876c63)
>    - **在线链路**：需先确认 AI 应用已接入 X-RAY，trace 数据在平台上可正常查看。→ 参考：[AI 应用链路埋点上报规范](https://docs.xiaohongshu.com/doc/ba8f8236ae557122bcc71af185d1ad82)

## 3. 创建任务：4 步配置流程

进入 **AI 应用评估 → 自动评估**，点击右上角「**+ 新建自动评估任务**」。

【截图：自动评估列表页，标注右上角「+ 新建自动评估任务」按钮 + 列表字段（名称/数据类型/执行周期/状态/时间范围/操作）】

---

### 第一步：基本配置

【截图：新建任务第一步，显示任务名称/评估类型（在线链路数据/离线数据集）/执行时间范围（周期+时间点）】

| 配置项 | 说明 |
|--------|------|
| **任务名称** | 建议体现业务场景和评估目标，例如「sug粗筛-在线链路-每日评估」，而不是团队内部缩写 |
| **评估类型** | 选择数据来源：**在线链路数据** 或 **离线数据集**。这个选择决定了第二步的配置路径，两者差异较大 |
| **执行时间范围** | 周期（按天/按周/按小时）+ 时间点，决定任务每次自动触发的时机 |

> **注意**：评估类型一旦选定，第二步的配置项会完全不同。在线链路和离线数据集是两条独立的配置路径。

---

### 第二步：评估数据

这是整条任务最关键也最复杂的一步，决定了**这次任务到底在评什么**。

---

#### 路径A：在线链路数据

在线评估时，需要先选择**评估方式**，不同方式决定了系统送给评估器的数据粒度和结构：

| 评估方式 | 适用场景 | 评估器限制 | 配置项差异 |
|---------|---------|-----------|-----------|
| **单 SPAN** | 评估某个中间环节（如某次 LLM 调用的输出质量） | LLM / HTTP 评估器均可 | TraceId筛选 + **SPAN筛选** + 链路采样率 |
| **全链路评估** | 评估整条链路的综合效果（基于所有 SPAN 的整体数据） | **只能用 HTTP 评估器** | TraceId筛选 + 链路采样率 |
| **结果评估** | 只评估最终 input / output，不感知中间过程 | LLM / HTTP 评估器均可 | TraceId筛选 + 链路采样率 |

【截图：第二步评估数据（在线链路），显示三种评估方式切换 + TraceId筛选（链路/日志条件）+ 链路采样率】

**TraceId 筛选**：用于过滤进入任务的链路范围，支持按链路属性（如 `name = agi`）或日志条件进行筛选，可多条件叠加。目的是让进入任务的样本"问题集中、边界清楚、规模可控"。

**SPAN 筛选**（仅单SPAN模式有）：在已筛选的链路范围内，进一步指定评估哪个具体的 SPAN（Observation），例如按 SPAN 名称、类型等过滤。

**链路采样率**：控制每次任务从符合条件的链路中抽取的比例（%），主要影响的是模型 token 消耗。如果业务每天链路量超过 10 万条，建议设置采样率，将每天进入评估的样本量控制在 1 万条以内，避免 token 成本失控。

---

#### 路径B：离线数据集

选择离线数据集路径时，先选定**目标数据集**（系统会自动展示数据集包含的字段，如 input / output），再决定是否开启**回放配置**。

【截图：第二步评估数据（离线数据集），显示目标数据集选择 + 数据集字段标签（input/output）+ 回放配置开关】

---

**情况一：不开启回放**

数据集里已经包含待评估的完整字段（比如 input + AI的实际输出 + 标准答案），可以直接进下一步，在第三步的字段映射中把数据集字段对应到评估器变量即可。

---

**情况二：开启回放**

如果数据集里只保存了**原始输入**（比如用户问题 + 标准答案），但没有 AI 应用的实际输出——需要先让系统把数据集里的每条 input 重新发给 AI 应用跑一遍，拿到回放结果，再用回放结果做评估。这种场景就需要开启回放配置。

【截图：第二步评估数据（离线数据集 + 开启回放），显示对象类型/Name/Domain/Path/Method/IsStream/请求Header/请求Body 等字段】

开启回放后，首先选择**对象类型**（即智能体平台），不同类型的配置流程有明显差异：

**选择 ALLIN 平台 / Darwin 平台**：直接从下拉列表选择你已创建或有权限的智能体，平台会自动填充接口信息，无需手动配置 Domain / Path / Header 等字段。

**选择自研 Agent（自定义模型）**：需要手动填写完整的 HTTP 接口信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| **Name** | 回放目标的名称，便于识别 | XPolit |
| **Domain** | 服务的域名 | `https://xray.devops.xiaohongshu.com` |
| **Path** | 接口路径 | `/aiagent/api/v1/chatbot/chat` |
| **Method** | HTTP 方法 | POST |
| **IsStream** | 是否流式响应 | 按实际接口情况选择 |
| **请求 Header** | 附加在回放请求里的 Header，例如标记数据来源（`X-LangFuse-Source: replay`）和关联的评估任务 ID | `X-LangFuse-Source: replay` |
| **请求 Body** | 从数据集字段提取值，组装成接口入参。支持 JsonPath 表达式，例如 `[{"content":"$.input[0]"}]` 表示把数据集的 `input` 字段第一条作为消息体 | `messages: [{"content":"$.input[0]"}]` |

> **理解回放的数据流向**：
> - 系统从数据集取 `input` → 按请求 Body 配置组装入参 → 发给目标接口 → 得到回放结果（replayResult）
> - 第三步字段映射时，`input` 和 `target`（标准答案）从数据集取，`out_put`（模型实际输出）从 `replayResult` 里用 JsonPath 提取（例如 `$.messages[?(@.type=='ai')].content`）

---

### 第三步：评估器

在这一步，将评估器挂到任务上，并完成**字段映射**。

【截图：第三步评估器，显示 tab 分组（每个tab对应一个评估器组）+ 评估器下拉选择 + 字段映射配置（数据集字段 → 评估器变量）】

**多评估器配置**：一个自动评估任务可以同时挂多个评估器，对同一批样本从不同维度打分（例如同时评估「相关性」和「准确性」）。页面上每个 tab 对应一个评估器，点击 + 号可以继续添加。

**字段映射**：告诉系统应该从当前数据来源中取哪些字段，送给评估器的哪些变量。这是最容易被低估的配置项，但它直接决定评估器最终看到的内容是否正确。

> **高频误区**：字段映射配错了，评估器看到的是空值或错误字段，导致结果全部异常——但错误原因不会在任务层面报出来，需要进实验明细的单条详情才能看到。

字段映射使用 **JsonPath** 语法从数据来源中提取字段值。JsonPath 语法参考：[JsonPath 文档](https://github.com/json-path/JsonPath)。

字段来源根据数据类型不同而不同：

**在线链路场景**：字段来源为链路 SPAN 的 input / output / metadata，使用 JsonPath 提取，例如：
- `$.data.input.messages[?(@.role == 'user')]` — 提取对话消息中用户的发言
- `$.data.input.raw_input` — 提取原始输入
- `$.data.output.response_content` — 提取模型输出内容

**离线数据集场景（不开启回放）**：字段来源为数据集字段（`dataset`），直接用 `$.input`、`$.output` 等路径取值。

**离线数据集场景（开启回放）**：字段来源有两种可以混用：
- `dataset`：数据集原始字段，用于取 input（原始问题）和 target（标准答案）
- `replayResult`：回放后返回的响应体，用于取模型实际输出，例如 `$.messages[?(@.type=='ai')].content`

---

### 第四步：数据回流（可选）

数据回流不是每条任务都必须开启，但它是**形成评估闭环**的核心能力——把低分/高分样本自动沉淀回数据集，供下次离线回归使用。

【截图：第四步数据回流，显示启用开关 + 目标数据集 + 数据集版本策略 + 回流条件（评估器+score操作符+阈值）+ 字段映射（JsonPath提取链路字段到数据集字段）】

| 配置项 | 说明 |
|--------|------|
| **启用数据回流** | 开关，默认关闭 |
| **目标数据集** | 符合回流条件的样本写入哪个数据集 |
| **数据集版本策略** | 自动创建新版本，便于按周期追踪样本变化 |
| **回流条件** | 选择评估器 + 操作符（>=、<=、= 等）+ 阈值，例如「sug粗筛，score >= 2」表示高分样本回流 |
| **字段映射** | 用 JsonPath 从链路字段中提取值，写入数据集对应字段，例如 `$.data.input.messages[?(@.role == 'user')]` → 数据集字段 `messages` |

回流条件支持**多评估器组合**和 **and / or 逻辑**，可以表达复杂的筛选规则，而不只是简单的"低于多少分就回流"。

## 4. 运行和管理任务

任务创建完成后，后续重点从"怎么配"转到"怎么跑、怎么管、怎么看结果"。

【截图：自动评估列表页，标注各操作列：查看/触发/编辑/删除 + 状态开关】

| 操作 | 说明 |
|------|------|
| **触发** | 立即执行一次评估，不等定时调度。适合刚创建完任务、调整完评估器后做快速验证 |
| **查看** | 进入任务详情，查看基础信息和历史实验列表 |
| **编辑** | 重新打开 4 步配置页，可以更换评估器、调整筛选条件、修改采样率或补充回流规则 |
| **状态开关** | 控制任务是否继续参与周期调度。暂停中的任务不会自动触发，但历史实验记录保留 |
| **删除** | 删除任务及其配置（实验记录需另行处理） |

## 5. 常见问题

**在线链路和离线数据集怎么选？**
如果要持续监控线上效果、发现实际用户请求中的问题，选在线链路；如果要做版本回归、验证特定场景、用已知 bad case 测试，选离线数据集。两者也可以建两条任务并行跑。

**三种评估方式（单SPAN/全链路/结果评估）怎么选？**
- 想评某个中间步骤（如 RAG 检索质量、某次 LLM 调用）→ **单SPAN**
- 想评整条 Agent 链路的综合表现 → **全链路评估**（**注意：必须配合 HTTP 评估器使用**，LLM 评估器不支持此模式）
- 只关心最终给用户的结果对不对 → **结果评估**

**字段映射配完怎么验证是否正确？**
触发一次任务，等实验跑完后进实验明细，找一条样本点「详情」，查看「评估器入参」里的字段值是否符合预期。如果字段为空或值不对，回来检查 JsonPath 表达式。

**数据回流配置了但没有数据进来，怎么排查？**
检查两点：回流条件的分数阈值是否合理（如果条件太严，可能确实没有样本满足）；字段映射是否正确（如果字段取值为空，样本即使满足分数条件也可能无法写入）。
