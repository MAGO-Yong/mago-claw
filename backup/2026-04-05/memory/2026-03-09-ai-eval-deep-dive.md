## 5. 评估器模块 - 自定义评估规则

### 5.1 概念

**评估器** = 自定义打分/评分逻辑

### 5.2 可能的评估指标类型

```
1. 准确性指标 (Accuracy)
   - Exact Match: 完全匹配答案
   - BLEU Score: 短语匹配度
   - ROUGE Score: 摘要评估

2. 关联性指标 (Relevance)
   - 与问题的相关度

3. 流畅性指标 (Fluency)
   - 回答的自然程度

4. 成本指标 (Cost)
   - Token消耗量
   - API调用成本

5. 延迟指标 (Latency)
   - 响应时间
   - P95/P99延迟
```

### 5.3 评估器配置界面预期

```
创建评估器
├─ 评估器名称
├─ 评估类型（选项）
├─ 打分规则
│  ├─ 权重配置
│  ├─ 阈值设置
│  └─ 计算公式
├─ 适用场景标签
└─ 预览/测试
```

## 14. 浏览器交互挑战

### SPA 框架复杂性
该应用使用高度反应式的前端框架（可能是 React/Vue），导致：
- 元素生命周期极短，快照中的 ref 很快过期
- 传统 TAB 点击导航困难
- URL 参数（`activeTab=201/202` 等）不直接控制视图切换

### TAB 导航参数推测
从现有 URL 和菜单结构推测：
- `activeTab=201` → 数据集 (Datasets)
- `activeTab=202` → 自动评估 (Auto Evaluation)
- `activeTab=203` → 评估实验 (Evaluation Experiments)
- `activeTab=204` → 评估器 (Evaluators)
- `activeTab=205` → LLM 配置 (LLM Configuration)

### 解决方案
需要通过以下方式突破界面限制：
1. 等待 SPA 完全渲染后延迟操作
2. 使用 MutationObserver 监听 DOM 变化
3. 利用 React DevTools/Vue DevTools 进行状态管理
4. 通过浏览器 API 获取 Shadow DOM 内容
