## OpenClaw 故障排查（上午）

用户的 OpenClaw Gateway 启动失败，完整排查过程：

1. **根本原因**：`libsignal` 模块缺失（`/usr/lib/node_modules/openclaw/node_modules/libsignal/src/curve.js` 不存在），导致 Gateway 启动即崩溃（exit 1，循环重启 20+ 次）
2. **修复过程**：
   - npm 重装时遇到 SSH 权限问题（`git@github.com: Permission denied`），通过 `git config --global url."https://github.com/".insteadOf` 解决
   - 目录残留问题（ENOTEMPTY），手动清理 `/usr/lib/node_modules/.openclaw-gX1GdeX9` 和 `/usr/lib/node_modules/openclaw` 后重装
   - 重装成功：`added 675 packages in 6m`
3. **飞书卡片 HTTP 400**：Gateway 启动后飞书 streaming 发送失败，在配置中加入 `channels.feishu.streaming: false` 解决，config hot reload 自动生效无需重启
4. **AI 死循环问题**：飞书里的 AI 在处理图片时陷入死循环重复输出，与 streaming 问题相关，关闭后消失
5. **API 限流**：死循环消耗大量 API 调用，后续触发 Kimi API rate limit，临时限流，自动恢复
6. **版本升级**：从 2026.3.8 升级到 2026.3.11（最新），升级前先配置 git HTTPS，升级成功
