## 一、评测背景

本次评测目标，是对比同一批 X-RAY 可观测 Skill 在两个不同 Agent 引擎中的实际效果差异：

- **open-claw**：基于 Claude 的通用 Agent 平台，通过读取 Skill SKILL.md 文件调用能力
- **自研 xray-agent**：内部研发的专用 Agent，针对 XRay 可观测平台场景定制

**核心前提**：两边使用的是同一套 X-RAY Skill，暂不把底层模型差异作为主要归因对象，重点从真实使用体验出发做对比。

### 评测维度（共 7 项，满分 70 分）

| 维度 | 权重说明 |
|------|----------|
| 响应速度 | 从发问到给出结果的耗时感知 |
| 意图识别 | 是否正确理解用户意图和目标对象 |
| Skill 选择 | 是否命中正确 Skill，路径是否合理 |
| 推理过程 | 中间过程的合理性与分支处理能力 |
| 结果完整性 | 最终答案是否完整，有无截断/丢失 |
| 结果准确性 | 关键信息是否准确，有无幻觉 |
| 交付体验 | 最终展示是否清晰可读，可直接使用 |

---

## 二、评测方法

- 两个 Agent 使用完全相同的题面
- 按真实用户使用方式发问，不额外提示"该调用哪个 Skill"
- 每题重点记录：Skill 命中情况、意图识别、最终结果是否稳定收敛、是否出现展示异常
- 对 xray-agent，除页面展示外，额外检查底层网络请求与 SSE 事件流，避免把"前端没渲染"误判成"Agent 完全没执行"

### 题目覆盖范围

| 轮次 | 场景类型 |
|------|----------|
| 第一轮 | 日志查询、trace 搜索、指标分析、告警事件、推理判断、多 Skill 串联 |
| 第二轮 | 告警规则搜索、告警规则详情、服务树查询（进行中） |

## 五、综合分析

### 5.1 整体态势

**open-claw** 整体处于领先地位，但优势主要体现在"稳定交付"，而非"思路更优"。

**xray-agent** 的问题更多是工程层面，思路和规划能力已有基础，但被前端渲染、流式稳定性、执行链路中断系统性拖累。

### 5.2 两边的核心差异

| 维度 | open-claw | xray-agent |
|------|-----------|------------|
| 结果稳定性 | ✅ 高 | ❌ 低（多次交付失败） |
| 最终答案质量 | ✅ 完整，可直接使用 | ⚠️ 有潜力，但经常未能展示 |
| 推理/规划意识 | ✅ 较好 | ✅ 不差，部分题更优 |
| 分支/降级处理 | ⚠️ 一般 | ✅ Q8 表现突出 |
| 体验可读性 | ✅ 干净，交付感强 | ❌ 噪音多，过程卡片干扰大 |
| 工程稳定性 | ✅ 基本稳定 | ❌ SSE 断流、network error 多次出现 |

### 5.3 xray-agent 问题归因

从多轮评测来看，xray-agent 的问题**不只是模型能力问题**，是以下几层叠加：

1. **Agent runtime 执行链路较长**，出错概率随链路变长而累加
2. **前端流式渲染承接不稳定**，长结果或多步骤结果未能稳定落地
3. **SSE 连接稳定性不足**，出现 incomplete chunked read / network error
4. **会话状态残留**，旧对话内容、skill 列表混入当前页面，影响展示纯净度

很多题它可能"会做"，但"没完整交付出来"。

## 六、优化建议

### 对 xray-agent（按优先级）

| 优先级 | 问题 | 建议 |
|--------|------|------|
| P0 | 流式渲染/SSE 稳定性 | 修复 incomplete chunked read，增加断线重连机制 |
| P0 | 最终结果落地 | 增加"最终摘要兜底"，不能只展示 task/execute 过程卡片 |
| P1 | 会话隔离 | 避免旧对话内容、Skill 列表污染当前会话展示 |
| P1 | 降级链路补全 | Q8 的 PQL→服务告警降级逻辑值得推广到其他 Skill |
| P2 | 页面 reload 路由 | 修复 reload 后默认回退 Sample Agent 的问题 |

### 对后续评测

- 继续分两类题：**原子能力题**（单 Skill 命中率）+ **多 Skill 串联题**（链路稳定性）
- 增加**稳定性复测**：同题重复执行 2-3 次，观察结果一致性
- 关注"题型分化"：告警规则详情、分支判断类题目是 xray-agent 的潜力方向

## 七、总结

### 第一轮结论（Q1-Q6）

> **open-claw 5 胜，xray-agent 1 胜，平均分 61.5 vs 51.0**

open-claw 当前整体交付效果更稳定，尤其在复杂排障链路题上优势明显。

### 第二轮阶段性结论（Q7-Q8）

> **1:1 平局，但题型分化出现**

xray-agent 在规则详情、分支降级类题目上展现出更好的推理能力，问题主要集中在工程稳定性。

### 综合判断

**现阶段**，如果从"用户能否稳定拿到一份可用答案"这个角度评估，**open-claw 明显优于自研 xray-agent**。

**潜力来看**，xray-agent 的思路和规划能力已经有基础，如果能解决流式稳定性和结果落地问题，在部分专项题型上有超越 open-claw 的潜力。
