### AI Native 化讨论关键结论
1. **节点1（SKILL上传）**：暂不改，上传是用户主动行为，Agent 无法主动判断哪些 SKILL 需要绑定规则
2. **节点2（绑定闭环）**：绑定完成后 Agent 给完整闭环反馈——推到哪个群、历史成功率；并可预览「告警触发后会发生什么」
3. **节点3（诊断报告）**：
   - 新增「推理过程」折叠块（6步：查日志→排GC→分析下游→查变更→历史匹配→综合判断）
   - Chip 标注「根据根因动态生成」而非固定选项
   - 侧边栏有完整诊断结论 + 推理透明 + 文件报告链接
4. **节点4（天级日报）**：从数据展示改为「Agent 判断在最上面 + 行动项 + 数据压到底部」，每个行动项有具体按钮和 Agent 回复

### 告警卡片设计决策
- 告警卡片包含完整字段：服务名、告警规则、触发条件（表达式/持续/当前值）、三个时间、认领提示、静默、操作链接
- 底部诊断状态条：「诊断中（loading）」→「已诊断（根因+置信度）」
- 诊断完成后整张卡片可点击（hover 蓝色边框），点击打开右侧侧边栏
- 静默/操作链接区域 stopPropagation，不误触发侧边栏
- 去掉「接着聊」按钮，点卡片整体触发

- 确认定位：**Claw 只负责 A 链路（配置阶段）**，B/C 链路（Hi IM 群推送+诊断）完全不变
- 两个 Tab：
  - **Tab1 流程对比图**：现状（4步痛点）vs 新方案（4步体验），A0-A4 步骤卡片，4个关键依赖
  - **Tab2 Claw 对话模拟**：完整可交互对话（安装→感知→选SKILL→选规则→闭环预期），右侧实时步骤说明

#### 4. 关键依赖（Claw 方案）
- ClawHub 需发布「XPILOT 告警诊断」SKILL
- XPILOT 需开放 SKILL 上传 API + 规则绑定 API
- 现有 alarm-rule-search SKILL 可直接复用
- 需获取「用户负责哪些服务」（服务树/oncall 系统）

### 设计决策沉淀
- **Claw 不替代 XPILOT**：Claw 是操作入口，XPILOT 是执行引擎和 SKILL 中心
- **Hi IM 群是告警推送主战场**：因为是多人协作场景，Claw 是个人补充入口
- **智能预选**：Claw 根据规则名称和 SKILL 描述自动预选，减少用户判断负担
