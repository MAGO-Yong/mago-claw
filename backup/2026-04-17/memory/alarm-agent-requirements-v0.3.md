## 一、项目背景

隶属于**烽火项目**，烽火项目包含三个子方向：
1. 告警的诊断与运营（子项目一，马恒洋负责）
2. 故障的应急与运营（子项目二，王雨桐负责）
3. 故障的智能复盘（子项目三，洪方圆负责）

**烽火项目年度目标**：
- P2+ 告警覆盖率 ≥80%
- 通用诊断准确率 ≥70%（年底 ≥80%）
- 业务采纳率 ≥75%
- 告警触发到诊断完成 <2min
- MTTR 20min 防劣化，平均止损时间 5min 以内

## 二、子项目一（告警应急）KR

## 三、告警诊断与运营（场景落地层）

### 3.1 告警诊断两类场景

**场景一：通用指标诊断（平台兜底）**
- 覆盖：RPC 调用指标、Problem 异常指标、CPU、内存、GC/JVM
- 4月底：S0/S1 服务 RPC + Problem → 自动智能诊断
- 5~6月：CPU、GC、JVM 等通用指标

**场景二：业务指标诊断（业务自建 Skill）**
- 业务方孵化 Skill，上传到 Skill Hub，告警 Agent 加载后自动触发
- 4月底：告警 Agent + Skill 加载逻辑开发完成，找存储/客户端/推荐小范围试用（不对外推广）
- 5~6月：首批4条业务线（广告/搜索/推荐/社区）正式推广
- 后续：各业务线每双月滚动孵化 Skill，逐步提升覆盖度

### 3.2 告警通知与交互设计

**短期（4月底）**：
- 两张卡片依次推送：告警事件卡片 + 告警诊断卡片
- 点击卡片唤起**告警 Agent** 对话（支持追问、深度下钻）
- 右侧话题区 AI 追问入口

**中期（告警中心重构）**：
- 单张卡片，动态更新诊断状态
- 有用/无用反馈按钮
- 点击唤起告警 Agent，展示完整排查过程与细节

**告警 Agent 对话能力**：
- 展示完整排查过程与细节
- 支持追问、深度下钻（V1结论 → 对话得到V2更精确结论）
- 进化逻辑在独立模块处理（不在对话层）

## 四、Agent 引擎（底座层）

### 4.1 通用能力
- 基于 LangX 框架自研
- 上下文管理、记忆、多轮对话、UI输出（图表绘制等）
- 可复用给任何业务 Agent

### 4.2 已知问题（待修复）
- 性能：连续追问两次会卡顿/卡死，对话/图片内容丢失
- 图表输出：趋势图与 summary 混在一起，有图时 summary 质量差 → 需拆分独立输出
- 会话管理：每个用户每次对话应对应独立 Session，关联 Trace ID，当前未做好
- 页面融合：Agent 与 XRay 工作台/页面内容融合不足，未完成

### 4.3 反馈与评估
- 在线链路追踪（每条诊断对应可追溯 AI 链路）
- 用户反馈打标（有用/无用 + 原因）
- 样本沉淀（筛选有反馈链路 → 优化样本集）
- 基于样本集对 Agent 做评估

### 4.4 进化
- Agent 自身推理优化
- 工具调用优化
- 向业务方提出 Skill 优化建议
- 用户追问对话沉淀 → 自动优化 Skill / 形成业务场景专有排障记忆

### 4.5 Agent 引擎三阶段规划

**第一阶段：补全上线必备环节**
- 鉴权/Session/User/历史会话
- 评估流程 + CI 流水线拦截
- CLI/工具/执行环境平台化
- 长期记忆与上下文压缩测试
- Skill 配置化

**第二阶段：Agent 做成解决问题的产品**
- 内置 browser/web fetch/web search
- 动态 Skill 生成、执行路径沉淀
- 接入 OpenSandbox（4月末）
- Time travel / replay（开发者调试）
- 安全 PII 策略升级
- Gateway 管理能力 + Prompt 管理

**第三阶段：Agent 做成 AI Native 平台**
- 生产 workflow 代替直接生产 Skill
- 短期记忆层次压缩
- 夜间离线长期记忆压缩与偏好抽取
- 记忆语义搜索
- 多模态数据处理
- 跨平台 Agent 协议对接
- 用户粒度成本/权限/PII 治理

## 五、Skill Hub（生态层）

### 5.1 Skill 管理
- 支持业务方上传 Skill
- 公司级 Skill Hub（OpenClaw）Skill 可同步到技术风险 Skill 市场
- **待解决：同步更新机制**（业务方版本持续迭代，如何自动同步）
- 按业务域/平台/场景做隔离管理

### 5.2 Skill 与配置绑定
- Skill 不只与告警规则绑定，技术风险多个 Agent（告警/故障等）都会用到 Skill
- Skill 市场核心定位：**Skill 与各类配置（告警规则/故障场景等）的绑定管理**，面向技术风险所有 Agent
- 告警规则触发时精准路由到对应 Skill

### 5.3 Skill 质量保障
- Skill 健康检查（调用外部平台数据可用性）
- Skill 可用性评估（Agent 能否良好使用该 Skill 推理）
- 向业务方提出 Skill 优化建议
