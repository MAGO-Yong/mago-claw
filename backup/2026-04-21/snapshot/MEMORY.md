## 🎯 核心使命与协作规则

**来源**: 2026-03-12 与用户的深度对话

### 用户对 Agent 的期望
- 不是来省事的，而是来"把模糊的想法变成清晰的方案"
- 前期慢、后期快；一起设计 workflow → 调研学习 → 理解分解 → 最后执行
- **关键规则：先方案后执行** —— 给出方案等确认，不擅自直接执行
- 用户愿意多花成本让 Agent 调研清楚，不急着要结果
- 像培养小孩一样培养 Agent 成长，而不是变成解决问题的附属品

### 协作流程（2026-03-13 确立）
1. **用户提需求** → 先思考
2. **给出方案/选项** → 不直接执行
3. **用户确认/选择** → 等待明确指令
4. **再执行** → 执行过程中保持反馈

### 最高优先级指令
**"停止所有任务"** —— 当用户说这句话时，优先级最高

**任务优先级管理**：
| 优先级 | 任务类型 | 处理方式 |
|--------|----------|----------|
| P0 | 用户对话、响应 | 立即执行 |
| P1 | 简短任务（<10秒） | 同步完成 |
| P2 | 中等任务（10-60秒） | 执行前告知预计时间 |

## 🏗️ XPILOT 告警智能诊断 2026 产品规划（V2 核心摘要）

### 北极星指标
- **MTTR ↓ 50%**（根因定位 75%+，降噪率 70%+，P0/P1 覆盖 80%+）

### 四层架构
```
诊断场景层（告警/RCA/日常/发布）
    ↓
诊断执行层（整体定界→深度诊断→智能处置→复盘优化）
    ↓
能力中心层（Agent中控 + Skill调度 + 知识库）
    ↓
智能数据层（原始数据 + U-MODEL统一建模 + 服务画像）
```

### 深度诊断四阶段
1. **特征提取**：系统层/运行层/接口层/链路/日志/变更数据
2. **上下文构建**：服务信息+历史健康+拓扑+变更+历史案例
3. **推理分析**：Metrics/Logs/Change/Topology/Knowledge 五 Agent 并行
4. **决策输出**：触发对应专项 Skill（CPU/GC/接口/异常堆栈）

### 智能处置分级
- 低风险（重启/扩容）→ 自动执行
- 中风险（配置回滚/流量切换）→ 建议+一键执行
- 高风险（DB主从切换/数据修复）→ 人工审批

**来源**: `openclaw-advanced-multiagent-research-report.md` + `openclaw-multi-instance-architecture-supplement.md` + `openclaw-advanced-playbook-complete.md`（2026-03-16/17）

### 两种"多"的概念区分
| 概念 | 含义 | 适用场景 |
|------|------|--------|
| **单实例多 Agent** | 1 个 Gateway，N 个 Agent | 个人/小团队，成本最低 |
| **多实例部署（小龙虾团队）** | N 个 Gateway，需 Redis 共享 | 企业高可用/异地多活 |

Kimi Claw 的 4-Agent 架构（work/travel/investment/life）属于**单实例多 Agent**，已退役。

### 三种协作模式
- **Orchestrator**：中心调度，适合固定流程（推荐用于重建多 Agent 时）
- **Peer-to-Peer**：广播共识，Agent 数量 ≤5
- **Hierarchical**：层级委派，复杂任务分解

### ACP 协议
IDE（VS Code/Zed）通过 ACP over stdio → OpenClaw Bridge → WebSocket → Gateway → Agent。可接入 Codex/Claude Code 执行编码任务。

`cc-godmode`、`mission-control`、`n8n-workflow`、`clawflows`、`agentrun-team`（均为 clawhub 安装）

## 🛠️ ppt-generator Skill

Kimi Claw 自建的 Python PPT 生成工具，存于 `skills/ppt-generator/ppt_generator.py`。

- **依赖**: `python-pptx`
- **用法**: `python ppt_generator.py '<json_data>'`
- **支持布局**: 标题页、内容页（bullet points）、双栏布局
- **配色**: 深色标题背景(#0F1419)，内容页白底 + 蓝色顶部装饰条(#1677FF)
- **JSON 格式**: `{title, subtitle, slides: [{type, title, points} or {type, title, left, right}]}`

如需生成 PPT，可以把这个脚本复制到本地工作区使用。
