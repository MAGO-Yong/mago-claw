# X-RAY Skill 双 Agent 第一轮评测报告

> 评测时间：2026-03-30
> 参与方：open-claw vs 自研 xray-agent
> 题目数量：Q1-Q6（含 Q3 复测）

---

## 1. 评测背景

本轮评测目标，是对比同一批 X-RAY 可观测 Skill 在两个不同 Agent 引擎中的实际效果差异：

- **open-claw**
- **自研 xray-agent**

评测前提是两边本质上使用的是同一套 X-RAY Skill，暂不把底层模型差异作为主要归因对象，而是从真实使用体验出发，重点评估：

- 响应速度
- 意图识别
- Skill 选择与调用路径
- 推理过程
- 结果完整性
- 结果准确性
- 最终交付体验

本轮属于第一轮冒烟评测，选取 6 道代表性题目，覆盖日志、trace、指标、告警、接口切入判断、多 skill 串联分析等典型场景。

---

## 2. 评测方法

- 两个 Agent 使用完全相同题面
- 按真实用户使用方式发问，不额外提示"该调用哪个 skill"
- 每题重点记录：命中正确 skill 与否、意图识别、能否收敛最终结果、是否出现中断或展示异常
- 对 xray-agent，除页面展示外，额外检查底层网络请求与 SSE 事件流，避免把"前端没渲染"误判成"Agent 完全没执行"

## 4. 总体结论

**第一轮整体结论：open-claw 整体领先**

- **open-claw** 优势主要体现在"稳定交付最终结果"
- **xray-agent** 在任务拆解、planner 意识上有一定亮点，但被前端渲染和流式稳定性明显拖累
- 很多题里，xray-agent 不是完全不会，而是"后端在跑、前端没完整交付出来"

**第一轮战绩：open-claw 5 胜，xray-agent 1 胜**

- 起手规划正确，后端也确认启动执行

## 7. 关键发现

### 7.1 open-claw 的核心优势

- 能更稳定把 Skill 执行结果收敛成最终答案
- 输出更像面向用户的"交付物"
- 在复杂链路题、多 Skill 串联题上明显更成熟

### 7.2 xray-agent 的优势

- planner 意识不错，起手拆解思路常常合理
- 在结构化信息任务（告警事件题）上有潜力
- 过程透明度较高，执行步骤可观测性好

### 7.3 xray-agent 的核心短板

前端渲染稳定性不足，流式结果展示不完整，多题出现"后端已执行、前端未完整交付"。

这直接拉低了：响应速度感知、结果完整性、用户可用性评分。

---

## 8. 问题归因

从第一轮来看，xray-agent 当前的主要问题**不只是模型能力问题**，更像是以下几层叠加：

1. Agent runtime 执行链路较长
2. 前端流式渲染承接不稳定
3. 长结果或多步骤结果未能稳定落地
4. 部分会话状态残留影响展示纯净度

换句话说，很多题它可能"会做"，但"没完整交付出来"。

---

## 9. 优化建议

### 对 xray-agent（优先级排序）

1. **P0：修复流式渲染和最终消息落地问题**
2. 对长链路分析增加"最终摘要兜底"机制
3. 避免只展示 skill/task/execute 过程卡片而缺少最终结论
4. 优化会话上下文隔离，减少旧内容污染

### 对后续评测

- 第二轮继续分两类题：原子能力题 + 多 Skill 串联题
- 增加稳定性复测（同题重复执行，观察波动）

---

## 10. 最终结论

> **open-claw 当前整体交付效果更稳定，尤其在复杂排障链路题上优势明显。**
>
> **xray-agent 在任务拆解和规划思路上已有一定潜力，但当前工程稳定性明显影响真实体验。**
>
> 现阶段如果从"用户能否稳定拿到一份可用答案"这个角度评估，open-claw 明显优于自研 xray-agent。
