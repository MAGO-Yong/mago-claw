- 结构在信息足够时才浮现（渐进式）

### v5 设计稿（diagnosis-v5-updated.html）

在 v4 基础上，基于马恒阳提出的 5 条优化意见，重写了 diagnosis-v5-updated.html：

- 从直接问字段改为：自我介绍 + 能力说明 + 开放式场景引导
- 确认文案：「先说说你想为哪个业务排障场景创建 Skill？简单描述一下就行。」

**② 触发方式多选 + 数据前置**
- 触发类型改为可多选 UI（告警触发/定时巡检/用户反馈/主动发现）

**③ 排查链路：图承载不确定点**
- 用户描述完 SOP 后，AI 立刻渲染流程图（不追问）
- 不确定的串并行关系以 ⚠️ 标注在图下
- 用户对着图说话修正，流程图实时更新

**④ 节点数据绑定：用户主导**
- 触发节点后，AI 告知可关联的数据类型（指标/链路/日志）+ 简短说明
- 用户自己输入：类型 + 大盘名/图表名或链接
- AI 确认 + 自动匹配原子 Skill + 流程图节点实时变绿
- 改掉了之前「AI 推列表，用户被动选」的交互

**⑤ Skill 生成后验证 + 迭代优化（已记录，未实现）**
- 生成后引导用户手动触发验证效果
- 支持多轮对话持续优化 Skill（说话改节点、加步骤、换图表）
- 核心理念：Skill 是迭代出来的，第一版够用就行
