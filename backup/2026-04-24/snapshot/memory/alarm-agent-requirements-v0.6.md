## 整体架构框架

本文档描述的是一套可横向扩展的 Agent 平台架构。当前落地第一个业务场景：**告警诊断与运营**。未来可持续扩展新的业务场景（服务巡检、变更管控、故障应急等），底座不变。

```
业务场景层（可横向扩展）
┌─────────────────┐  ┌──────────────┐  ┌──────────────┐
│  告警诊断与运营   │  │   服务巡检    │  │   变更管控    │  ...
│  └── 告警 Agent  │  │ └── 巡检Agent │  │ └── 变更Agent │
└─────────────────┘  └──────────────┘  └──────────────┘
         │                   │                  │
         └───────────────────┴──────────────────┘
                             │
              ┌──────────────┴──────────────┐
              │        Agent 引擎（通用底座）  │
              └──────────────┬──────────────┘
                             │
              ┌──────────────┴──────────────┐
              │    Skill Hub（生态层）         │
              │  触发源配置 × Skill 绑定管理   │
              └─────────────────────────────┘
```

**核心抽象：触发源 + 路由绑定**

| 业务场景 | 触发源 | 绑定对象 | 路由到 |
|---------|--------|---------|--------|
| 告警诊断 | 告警事件 | 告警规则 | 对应 Skill |
| 故障应急 | 故障单 | 故障类型/业务场景 | 对应 Skill |
| 服务巡检 | 定时任务 | 服务/指标 | 对应 Skill |
| 变更管控 | 变更事件 | 变更类型 | 对应 Skill |

## 一、项目背景

隶属于**烽火项目**，烽火项目包含三个子方向：
1. 告警的诊断与运营（子项目一，**马恒洋**负责）
2. 故障的应急与运营（子项目二，**王雨桐**负责）
3. 故障的智能复盘（子项目三，**洪方圆**负责）

**烽火项目年度目标**：
- P2+ 告警覆盖率 ≥80%
- 通用诊断准确率 ≥70%（年底 ≥80%）
- 业务采纳率 ≥75%
- 告警触发到诊断完成 <2min
- MTTR 20min 防劣化，平均止损时间 5min 以内

## 二、子项目一（告警应急）KR

- **KR1**：通用诊断能力建设，覆盖全量 S0/S1 Java/C++ 服务 P0/P1 告警，诊断准确率 ≥70% | **荀诩（韩奇祺）**
- **KR2**：业务排障规模化，5 条核心业务线（推搜广电商社区），每条业务线 Skill 覆盖率 ≥80% | **柯伦（孙宗尧）**
- **KR3**：诊断闭环，AI 建议采纳率 ≥60%，知识库沉淀案例 ≥100 条 | **荀诩 + 柯伦**
- **KR4**：告警运营，稳定性周报自动化覆盖率 ≥80% | **正一（马恒洋）**

## 三、告警诊断与运营（当前落地业务场景）

### 3.1 告警 Agent

**核心职责**：告警规则触发 → 产生告警事件 → 匹配路由 → 找到对应 Skill → 执行诊断 → 输出结论

**归属**：Agent 引擎团队（开发）+ 正一（产品）

**具体需求**：
- [ ] 接收 XRay 推送的告警事件（事件ID、告警规则ID、服务信息等）
- [ ] 根据告警规则ID在 Skill Hub 查找绑定 Skill，无匹配时走通用兜底 Skill
- [ ] 动态加载 Skill，传入告警上下文，执行诊断
- [ ] 将诊断结论格式化后推送到 Hi 卡片 / 告警中心

### 3.2 通用指标诊断（告警 Agent 覆盖范围之一）

**归属**：**荀诩（韩奇祺）**

**3.2.1 RPC + Problem 诊断（4月底）**
- [ ] 覆盖 S0/S1 Java 服务
- [ ] 支持 RPC 调用成功率/耗时告警、Problem 异常数量告警
- [ ] 调用 metric-query Skill 拉取指标趋势，识别异常时间点，关联上下游服务，给出根因判断
- [ ] 输出：异常概览 + 根因分析 + 处置建议

**3.2.2 CPU / GC / JVM 诊断（5~6月）**
- [ ] CPU 高：结合线程池、GC 情况综合判断
- [ ] GC 频繁：识别 Full GC / Young GC 异常，关联内存水位
- [ ] JVM 内存：Heap/Non-Heap 水位，关联 GC 趋势

**3.2.3 C++ 服务诊断（5~6月）**
- [ ] 建设初版 C++ 诊断能力，覆盖核心 S0/S1 服务

### 3.3 业务指标诊断（告警 Agent 覆盖范围之二）

**归属**：**柯伦（孙宗尧）**

**3.3.1 Skill 加载框架（4月底）**
- [ ] 告警 Agent 支持动态加载业务 Skill（从 Skill Hub 拉取）
- [ ] Skill 与告警规则绑定关系配置完成
- [ ] 小范围试用：存储、客户端、推荐各接入 1 个 Skill，验证链路可通

**3.3.2 首批业务线正式推广（5~6月）**
- [ ] 广告、搜索、推荐、社区，每条 Skill 覆盖率 ≥50%

**3.3.3 后续滚动扩展（7月起）**
- [ ] 各业务线每双月滚动孵化新 Skill，年底 ≥80%

### 3.4 告警通知与交互（告警 Agent 输出形态）

**归属**：**正一**（产品）+ Agent 引擎团队（开发）

**短期（4月底）**
- [ ] 告警事件卡片：服务名、告警规则、触发时间、严重级别
- [ ] 告警诊断卡片：诊断结论、根因、处置建议
- [ ] 点击卡片唤起**告警 Agent** 对话窗口
- [ ] 右侧话题区 AI 追问入口

**中期（告警中心重构，5~6月）**
- [ ] 两张卡片合并，状态动态更新（告警中 → 诊断中 → 诊断完成）
- [ ] 卡片内嵌有用/无用反馈按钮
- [ ] 点击唤起告警 Agent，展示完整排查链路

**告警 Agent 对话能力**
- [ ] 展示完整排查过程（指标图表 + 文字分析）
- [ ] 支持追问深度下钻（V1 → 追问 → V2更精确结论）
- [ ] 进化逻辑独立处理，不在对话层耦合

### 3.5 告警运营（KR4）

**归属**：**正一（马恒洋）**
- [ ] 稳定性周报自动化：汇总 oncall 周期内告警事件、诊断结论、处置结果
- [ ] 个人 oncall 日报（8月 MVP）：每日自动推送告警概况
- [ ] 年底目标：自动化覆盖率 ≥80%

## 四、Agent 引擎（通用底座）

**归属**：Agent 引擎团队
所有业务 Agent 共用，不与具体业务场景绑定。

### 4.1 通用能力

- 基于 LangX 框架自研
- 上下文管理、记忆、多轮对话、UI输出（图表绘制等）
- 可复用给任何业务 Agent

### 4.2 三阶段规划

**第一阶段：上线必备（当前 / 4月底）**

上线基础能力：
- [ ] 鉴权 / Session / User / 历史会话
- [ ] Skill 配置化
- [ ] 评估流程 + CI 流水线拦截
- [ ] CLI / 工具 / 执行环境平台化

高优 Bug 修复：
- [ ] Session 管理：每次对话对应独立 Session，关联 Trace ID
- [ ] 连续追问卡顿/卡死，对话/图片内容丢失
- [ ] 图表与 summary 混排，有图时 summary 质量差 → 拆分独立输出

**第二阶段：做成解决问题的产品（5~6月）**

产品能力：
- [ ] 接入 OpenSandbox（4月末）
- [ ] 内置 browser / web fetch / web search
- [ ] 动态 Skill 生成、执行路径沉淀
- [ ] Time travel / replay（开发者调试）
- [ ] 安全 PII 策略升级
- [ ] Gateway 管理能力 + Prompt 管理
- [ ] 页面融合：Agent 与 XRay 工作台 / 页面内容融合 | **归属：Agent 引擎团队 + XRay 前端**

评估与反馈：
- [ ] 在线链路追踪，每条诊断对应可追溯 AI 链路 | Agent 引擎团队
- [ ] 用户反馈打标（有用/无用 + 原因）| 正一（产品）+ Agent 引擎团队（开发）
- [ ] 样本沉淀：筛选有反馈链路 → 优化样本集 | 荀诩 + 柯伦
- [ ] 基于样本集对 Agent 做评估 | 荀诩 + 柯伦

**第三阶段：做成 AI Native 平台（7月起）**

进化能力：
- [ ] Agent 自身推理优化 | Agent 引擎团队
- [ ] 工具调用优化 | Agent 引擎团队
- [ ] 向业务方提出 Skill 优化建议 | 柯伦
- [ ] 用户追问对话沉淀 → 自动优化 Skill / 形成业务场景专有排障记忆 | Agent 引擎团队

平台化：
- [ ] 生产 workflow 代替直接生产 Skill
- [ ] 短期记忆层次压缩
- [ ] 夜间离线长期记忆压缩与偏好抽取
- [ ] 记忆语义搜索
- [ ] 多模态数据处理
- [ ] 跨平台 Agent 协议对接
- [ ] 用户粒度成本 / 权限 / PII 治理
