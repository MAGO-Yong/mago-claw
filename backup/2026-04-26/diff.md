+## 本周新增规则（2026-W17）
+
+### UI 设计评审：区分"对象"再分析
+- 当对话中存在多个视觉对象（原图 vs demo vs 其他截图）时，用户说"从字体/颜色看"必须先确认指哪个
+- 2026-04-24 犯过的错：用户说"从字体的间距、颜色"，Agent 以为是评价自己做的 demo，实际指原产品页面
+- **规则**：当用户说"从 X 维度看/分析"但没有明确主语，且上下文有多个候选，直接确认"你说的是哪张图？"
+
+### UI 诊断原则：先拆后治
+- 用户不要"帮我做一个好看的"，而是"告诉我现有方案哪里不好、怎么改"
+- 诊断框架：字体大小层级 → 字重主次 → 间距亲疏关系 → 颜色视觉焦点 → 一句话总结
+- 核心诊断金句："所有元素处于同一视觉层级，用户眼睛无处落脚" — 适用于大多数 UI 可读性问题
+
+### OKR 规划：指标对齐是第一优先级
