## 2026-04-06

### [纠正] skill-evaluator 评估方式错误
- **错误**：初次评估 xray-log-query 时直接调底层脚本（query_logs.py），未通过 Agent 接入点
- **纠正**：通过 browser 访问 `https://xray-agent.devops.xiaohongshu.com/` 真实发 query
- **根因**：skill-evaluator SKILL.md 缺少"接入点确认"强制步骤
- **修复**：在 SKILL.md Phase 0 增加 Agent 接入点强制确认步骤（commit `a670d75`）

### [纠正] 报告格式未对齐参考文档
- **错误**：评估报告格式简陋，缺少：分布统计、关键发现高亮、综合评分汇总表、SKILL 归因
- **纠正**：对齐参考报告（ee9837e9b101d1939ef9f23f40f75b55）格式（commit `70a1481`）
- **影响**：所有后续评估报告需按新格式输出

### [纠正] 多轮对话中关键前提反复丢失
- **错误**：在年终奖期权置换计算中（~15轮对话），反复犯三个错误：(1) 忘记「年终奖独立计税」，错误合并计算；(2) 用回购价当置换成本而不是$21；(3) 对比基准线漂移（¥31.3万→¥28.3万→¥22.6万多个版本）
- **纠正**：多轮复杂计算类对话，必须在第一轮将关键前提（计税方式、成本价、对比基准）写入持久上下文或每轮复用，不能依赖短期记忆
- **根因**：Agent 在多轮迭代中每次重新计算时重新假设前提，而非复用已有共识；缺少「前提校验」步骤
- **用户原话**：「你是傻逼吗？你好好看看上下文，看看我有多少年终奖」
- **修复**：复杂计算场景（>3轮迭代），每轮开头必须列出「已知前提清单」与用户确认一致后再计算
