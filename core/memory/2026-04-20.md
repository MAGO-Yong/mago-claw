### 关键证据五大维度（本次确认）
1. 📊 **指标**：量化数据（CPU/QPS/错误率/趋势）— 必填
2. 🔄 **变更**：发布记录，AI 标注「最可疑」一条，其余折叠 — 有则展示
3. 🔗 **依赖**：下游服务、DB、中间件异常 — 有则展示
4. ⚠️ **异常**：异常类型 + 代码位置合并 — 有则展示
5. 💥 **影响**：范围、持续时间、严重程度 — 必填
6. 📋 **历史**：相似故障匹配 — 有则展示

> 现有卡片最大信息缺口：缺少「影响」维度，只告诉在哪里异常，没有量化影响面。

### Hi 卡片能力评估
用户提供了 Hi 卡片模板配置文档（fc20a576e4644409dc23ad14bc65d157），评估结论如下：

**✅ 完全可以实现**
- 标题行（header + theme + tags badge）
- 根因结论区（blockLayout + isReferenceMode 灰底）
- 关键证据各维度（leftRightLayout 左右布局）
- 「最可疑」橙色标签（text tags + color:orange）
- 触发/推送时间（leftRightLayout + text）
- 操作按钮（button + operatorGroupLayout）
- 反馈行 👍/👎（button + action.type: callback + visibilityCondition 控制状态）

**⚠️ 部分可以实现（有限制）**
- 「诊断结论」蓝色底色块：只有灰底（isReferenceMode），无法自定义蓝色背景色和边框
- 变更/影响维度的彩色左边线：卡片引擎不支持自定义边线样式，只能用 icon 或 text color 间接模拟
- 「引用原消息」按钮：属于消息发送接口层能力（quoteMessageId），卡片内无法主动触发

**❌ 无法实现**
- 侧边栏（卡片内无浮层/展开面板能力）
- 侧边栏内的对话追问输入框

**关键结论**：「展开完整诊断」只能做成跳转链接，指向 Web 详情页；反馈 👍/👎 通过 callback 完全可实现。

## QS AutoFlow 会议纪要 + 合作探讨（晚间）

### 会议背景
正一（马恒洋）和小庄（许小平）探讨 QS 平台 AI 化研发流程的合作可能性。

### 核心要点

**1. 目标与背景**
- 研发迭代流程 AI 化，提升效率、释放研发人力
- PM 转型参与研发，AI 处理 Bug 修复、前端微调等细碎工作
- 路径：需求收集 → PRD 生成 → AI 编码 → 测试验证 的自动化平台
- PM 和研发只负责关键环节审核与校验

**2. QS 平台定位**
- 提供集成 Claude Code 的在线研发环境（QSCode），7×24h AI 开发能力
- 标准化开发环境 + 预置 Skills + 原子能力 Tasks
- 用户通过 CLI 或平台界面发起任务
- **重点优化 CI/CD，使其更"AI 友好"**
- QS 作为通用能力提供者，CLI 能力可被其他业务团队复用构建自己的自动化流程
- QS 自身持续迭代发展新业务

**3. 关键挑战（双方共识）**
- **文档整合**：现有文档分散且非结构化，是 AI 理解的最大障碍 → 趋势是文档与代码库一体化管理，"代码即文档"
- **流程打磨**：各环节（需求分析/PRD生成/代码审查）需要反复打磨和验证
