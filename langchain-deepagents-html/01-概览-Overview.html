🚀 Deep Agents 概览

构建可以规划、使用子智能体（subagent），并利用文件系统处理复杂任务的智能体
原文：https://docs.langchain.com/oss/python/deepagents/overview

开始构建由大语言模型（LLM）驱动的智能体和应用的最简单方式——内置任务规划、用于上下文管理的文件系统、子智能体派发（subagent-spawning）和长期记忆能力。

你可以将 deep agents 用于任何任务，包括复杂的、多步骤的任务。

我们将 deepagents 视为一个「智能体容器（agent harness）」。它与其他智能体框架使用相同的核心工具调用循环，但内置了更多工具和能力。

deepagents 是一个独立库，构建在 LangChain 的核心智能体构建块之上。它使用 LangGraph 运行时来实现持久化执行、流式传输、人在回路（human-in-the-loop）等功能。

deepagents 仓库包含：

  • Deep Agents SDK：一个用于构建可以处理任何任务的智能体的软件包
  • Deep Agents CLI：一个基于 Deep Agents SDK 构建的终端编码智能体
  • ACP 集成：一个 Agent Client Protocol 连接器，用于在 Zed 等代码编辑器中使用 deep agents

LangChain 是一个框架，为你的智能体提供核心构建块。要了解更多关于 LangChain、LangGraph 和 Deep Agents 之间的区别，请参阅框架、运行时和容器（Frameworks, runtimes, and harnesses）。

创建 Deep Agent

Google

# pip install -qU deepagents langchain-google-genai
from deepagents import create_deep_agent

def get_weather(city: str) -> str:
    """获取给定城市的天气。"""
    return f"{city} 永远是晴天！"

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    tools=[get_weather],
    system_prompt="你是一个有帮助的助手",
)

# 运行智能体
agent.invoke(
    {"messages": [{"role": "user", "content": "sf 的天气怎么样"}]}
)

何时使用 Deep Agents

当你想构建能够执行以下操作的智能体时，使用 Deep Agents SDK：

  • 处理复杂的多步骤任务，需要规划和分解
  • 管理大量上下文，通过文件系统工具和摘要（summarization）功能
  • 切换文件系统后端，以使用内存状态、本地磁盘、持久化存储、沙箱（sandboxes）或自定义后端
  • 执行 shell 命令，通过 execute 工具在使用 沙箱后端 时
  • 委派工作给专门的子智能体（subagent）以实现上下文隔离
  • 跨对话和线程持久化记忆
  • 通过声明式 权限规则控制文件系统访问，限制智能体可以读取或写入哪些文件
  • 需要人工审批敏感操作，通过 人在回路（human-in-the-loop）工作流
  • 使用任何支持工具调用的模型——在前沿模型和开源模型之间 提供商无关（provider agnostic）

对于构建更简单的智能体，可以考虑使用 LangChain 的 create_agent 或构建自定义的 LangGraph 工作流。

核心能力

📋 规划与任务分解

Deep Agents 内置了 write_todos 工具，使智能体能够将复杂任务分解为离散的步骤、跟踪进度，并在新信息出现时调整计划。

✂️ 上下文管理

文件系统工具（ls、read_file、write_file、edit_file）允许智能体将大量上下文卸载到内存或文件系统存储中，防止上下文窗口溢出，并支持可变长度的工具结果。自动摘要功能会在上下文窗口变长时压缩较早的对话消息，使智能体在长时间会话中保持有效性。

💻 Shell 执行

使用 沙箱后端 时，智能体获得一个 execute 工具，可以运行 shell 命令来执行测试、构建、git 操作和系统任务。沙箱后端提供隔离，使智能体可以执行代码而不影响你的主机系统。

🔌 可插拔文件系统后端

虚拟文件系统由 可插拔后端 驱动，你可以根据用例进行切换。可选方案包括：内存状态、本地磁盘、LangGraph 存储（跨线程持久化）、沙箱（用于隔离代码执行：Modal、Daytona、Deno），或通过复合路由组合多个后端。你也可以实现自己的自定义后端。

👥 子智能体派发（Subagent Spawning）

内置的 task 工具使智能体能够派发专门的子智能体（subagent）以实现上下文隔离。这保持了主智能体的上下文干净，同时仍然可以在特定子任务上深入探索。

💾 长期记忆

使用 LangGraph 的 Memory Store 扩展智能体的跨线程持久记忆。智能体可以从之前的对话中保存和检索信息。

🔒 文件系统权限

声明 权限规则 来控制智能体可以读取或写入哪些文件和目录。子智能体可以继承或覆盖父级的规则。

👤 人在回路（Human-in-the-loop）

使用 LangGraph 的中断功能为敏感工具操作配置 人工审批。控制在执行前哪些工具需要确认。

🧩 技能（Skills）

通过可复用的 技能（skills）扩展智能体，提供专门的工作流、领域知识和自定义指令。

🪄 智能默认值

自带经过优化的系统提示，教会模型如何有效使用其工具——在行动之前先规划、验证工作、管理上下文。根据需要自定义或替换默认值。
