## 三、LOBI 好在哪里

### 3.1 速度碾压（最核心优势）

| 对比项 | LOBI | xray-agent |
|--------|------|-----------|
| 平均响应时间 | **82s** | **5-15min** |
| 超时题目数 | **0/17** | **2/17（Q13/Q18）** |
| 最快响应 | Q9（24s）、Q3（25s） | Q9（11s）、Q5（20s） |
| 最慢响应 | Q12/Q14/Q17（134s，约 2min） | Q12（20min）、Q15（25min） |

**结论**：LOBI 响应速度是 xray-agent 的 5-10 倍。xray-agent 之所以慢，主要因为：
1. 大量 `glob`/`ls` 文件系统探索（每次都重新找 Skill 文件）
2. 接口类型试错（Service/URL/Call 多次猜测）
3. metrics API 易超时无响应

LOBI 则通过预加载 Skills、路由规则优化规避了这些问题。

### 3.2 Q13/Q18 不超时（稳定性优势）

xray-agent 在 Q13（三维度分析，含 metrics）和 Q18（自主规划排查）上超时，因为 metrics-query-by-template 无响应。LOBI 在这两题均在 ~100s 内完成，说明 LOBI 要么对 metrics API 有超时兜底，要么走了不同的指标查询路径。

### 3.3 交互体验更好

- LOBI 在遇到 API 能力限制时主动说明（如 Q2："XRay API 只能查异常 Span，不能按耗时排序"），并给出替代建议
- xray-agent 有时会静默重试，用户无法判断是否在正常执行

### 3.4 Q1/Q2/Q3/Q14 表现更好

- **Q1**：LOBI 把 goodsnote vs goodsnote-live 分成两个独立子场景，xray-agent 有重叠
- **Q2**：LOBI 明确说明数据来源（异常 Span 视角）并主动解释 API 局限，xray-agent 直接给了可能误导的数据
- **Q3**：25 秒完成，简洁一句话结论，xray-agent 1m7s
- **Q14**：LOBI 完成告警规则+触发验证+日志三步，xray-agent 差不多但略慢

## 四、xray-agent 好在哪里

### 4.1 复杂推理题更深入

| 题号 | xray-agent | LOBI | 优势内容 |
|------|-----------|------|---------|
| Q5 | 64 | 53 | 推理题：xray-agent 逻辑更严密，LOBI 把题目识别为评测题（"这道题我认识"），元感知干扰了回答 |
| Q9 | 65 | 49 | 服务树查询：xray-agent 格式规范完整，LOBI 输出简略 |
| Q12 | 65 | 55 | 四段式分析：xray-agent 数据更充分，根因链路更完整 |
| Q16 | 62 | 55 | 服务归属+异常：xray-agent 异常归类更精准 |
| Q17 | 62 | 57 | 慢请求+Span+日志：xray-agent 三层分析更深 |

### 4.2 Q5 严重失分：LOBI 的元感知问题

LOBI 的 Q5 回答开头是："这道题我认识，这是评测 Q5 的题目——先从 trace 切入。"

这说明 LOBI 使用的是同一个 Main Session（有历史上下文），没有真正做到"每题新开会话隔离"。评测被上下文污染，Q5 的回答质量因此打折。

**这是 LOBI 评测数据的重要局限**：若能真正隔离每题的 Session，Q5 等题的得分可能更高或更低，目前数据有一定偏差。

### 4.3 意图识别更精准

xray-agent 在理解复杂多步指令时少犯错（8.8 vs 8.5），尤其在 Q5（推理题）、Q9（精准匹配服务树路径格式）上表现更好。

### 4.4 结果完整性更高（复杂题）

串联题（Q12/Q15/Q17）xray-agent 虽然慢，但输出更完整——因为它有更多时间做多轮工具调用和数据整合。LOBI 为了快速响应，部分题目的分析深度略浅。

## 六、综合结论

### 6.1 谁更好？取决于场景

| 场景 | 推荐 | 理由 |
|------|------|------|
| 简单/快速查询（单 Skill） | **LOBI** | 速度 5-10x，无超时风险 |
| 复杂串联排查（含指标） | **LOBI** | xray-agent metrics 超时，LOBI 能跑完 |
| 深度推理分析（四段式） | **xray-agent** | 推理更完整，数据更充分 |
| 上线生产使用 | **LOBI** | 稳定性高，用户等待时间可接受 |
| 评测/研究场景 | **xray-agent** | 推理过程可见，便于分析和改进 |

### 6.2 核心差距总结

**LOBI 比 xray-agent 好的地方（P0 优势）**：
1. ⚡ **速度**：82s vs 5-15min，差距 5-10x，这是用户感知最强的差异
2. 🛡️ **稳定性**：0 超时 vs 2 超时，metrics API 问题 LOBI 有兜底
3. 💬 **交互体验**：主动说明 API 限制，不让用户傻等

**xray-agent 比 LOBI 好的地方（P1 优势）**：
1. 🧠 **推理深度**：四段式、多维度综合分析更完整（Q12/Q17/Q5）
2. 📋 **输出格式**：服务树归属、分类统计格式更规范
3. 🔍 **意图识别**：复杂指令理解更精准

### 6.3 得分汇总

| 系统 | 可比 15 题总分 | 17 题总分 | 平均分 | 超时题 |
|------|--------------|---------|-------|-------|
| xray-agent | 874/1050 | 874（+2题超时） | 52.8/70（75.4%） | 2 |
| LOBI | 821/1050 | 935/1190 | 55.0/70（78.6%） | 0 |

> 结论：**整体（含超时）LOBI 领先**（78.6% vs 75.4%）；**可比 15 题 xray-agent 领先**（874 vs 821）。速度和稳定性是 LOBI 的核心优势，推理深度是 xray-agent 的核心优势。

## 七、改进建议

### 针对 xray-agent

| 优先级 | 问题 | 建议 |
|--------|------|------|
| P0 | metrics API 超时无兜底 | 增加 30s 超时限制，降级到其他数据源 |
| P0 | 每题重新 glob/ls 探索 | Skill 路径预加载，不重复探索 |
| P1 | 接口类型多次猜测 | 先查服务树确认接口类型，再调用 Skill |
| P1 | 无进度反馈 | 增加中间步骤提示（"正在查询 trace..."） |
| P2 | Q2 数据误导 | 明确区分"异常 Span 耗时"与"请求耗时" |

### 针对 LOBI

| 优先级 | 问题 | 建议 |
|--------|------|------|
| P0 | 上下文污染（Q5 元感知） | 评测时确保每题真正 new session 隔离 |
| P1 | Q9 服务树输出简略 | 强化格式输出规范 |
| P1 | 复杂题推理深度不足 | Q12/Q17 深度可以再挖掘 |
| P2 | Q4 慢请求查找失败 | 优化 xray-trace-search 慢请求查询能力 |
