# hi-assistant-profile

更新个人助理（机器人/bot）的基本信息，包括名称、头像和签名。

> 当前仅支持**修改**操作，暂不支持查询当前助理信息。如果用户想先查看现有信息再修改，请告知此限制。
>
> 如果不确定参数用法，可先执行 `bunx @xhs/hi-workspace-cli@0.2.12 assistant-profile:update-info --help` 查看。

## 修改助理信息

### 头像来源判断与处理

`--icon` 参数**只接受本地文件路径**。根据用户提供的头像来源，按以下规则处理：

| 用户提供的内容                             | 判断方式                        | 处理方式                                        |
| ------------------------------------------ | ------------------------------- | ----------------------------------------------- |
| 本地文件路径（如 `/Users/.../avatar.png`） | 以 `/` 或 `~` 或 `./` 开头      | 直接传给 `--icon`                               |
| 网络图片 URL（如 `https://...`）           | 以 `http://` 或 `https://` 开头 | **先下载到 `/tmp/`，再将本地路径传给 `--icon`** |
| 聊天中嵌入的图片（含 URL）                 | 从上下文提取图片 URL            | 同上，先下载再传入                              |

**下载网络图片的标准步骤**：

```bash
# 使用 curl 下载（-L 跟随重定向，-o 指定保存路径，使用时间戳避免文件名冲突）
curl -L -o /tmp/assistant_avatar_$(date +%s).png "<图片URL>"

# 确认文件有效（大小不为 0）
ls -l /tmp/assistant_avatar_*.png

# 然后将下载后的本地路径传给命令
bunx @xhs/hi-workspace-cli@0.2.12 assistant-profile:update-info --icon /tmp/assistant_avatar_<时间戳>.png
```

**下载后验证**：

- 检查文件大小是否为 0（空文件说明下载失败），若为 0 则向用户提示 URL 可能无效。
- 若 `curl` 返回非 0 退出码，不要继续执行，直接向用户反馈下载失败。

**图片要求**：

- 下载时保留原始文件后缀（如 `.png`、`.jpg`）；如果 URL 中无后缀，默认使用 `.png`。
- 超过 **30 MB** 的图片会**自动压缩**到 25 MB 以内（GIF 格式除外，不支持自动压缩）。
- 支持格式：jpeg、jpg、png、gif、webp。
