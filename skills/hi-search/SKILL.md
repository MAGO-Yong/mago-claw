# hi-search

技能核心能力通过 CLI 提供，在执行 CLI 命令时，**禁止**猜测命令用法：

```bash
bunx @xhs/hi-workspace-cli@0.2.12 search --help              # 所有命令概览
```

## 技能使用方式

- **「我」** → 先 `search:me` 取当前用户 contactId（邮箱）。
- **多步组合**：复杂意图拆步执行。群 → 消息：先搜群拿会话 ID，再用会话 ID 搜消息。意图模糊时按类型分别搜（员工、文档、群聊等）再汇总

## 信息查询

- 当用户通过线索、语义搜索，你可能难以通过单一业务数据帮用户找到预期的信息，你可以尝试通过多元数据来丰富回答的内容
- 当使用语义化搜索的时候，你需要尽量保留用户原本查询的意图，不要用关键词或缩略表达进行替换

## 分页

- 仅当 `hasMore === true` 且用户明确要更多时才翻页
- 无更多结果时如实告知

## 输出规范（严格遵守）

- **简洁输出**：当输出内容里包含数据的 ID 、邮箱、头像等信息的时候，除非用户明确需要，否则不展示给用户
