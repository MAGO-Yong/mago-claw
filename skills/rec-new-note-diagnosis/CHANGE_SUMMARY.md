# 诊断报告增强 - 变更摘要

## 核心改进

按照用户要求：**凡包含"异常"、"高/极高风险"等关键字的结论，都必须给出明确的详细信息**

## 改进内容

### 1. 新增 `AnomalyDetail` 数据类

用于结构化存储异常详细信息，包含：
- 严重程度、指标名称、当前/基准值
- 绝对差异、百分比变化
- 受影响服务/渠道、异常时间点
- 根因、关联变更、建议操作

### 2. Step 1 异常报告增强

当检测到异常时，现在输出：

```
【🔴 异常确认 - 详细信息】

【判定摘要】
├─ 异常指标:   1H新笔记占比
├─ 异常时间点: 2026-03-20 05:24:00
├─ 检测阈值:   下跌超过 5%
└─ 判定依据:   SOP Step 1.1 - 新笔记占比下跌超过阈值触发异常调查

【关键时间点数据】
  03-20 05:24: 1.28%
  03-20 05:24(昨天): 2.02%
  跌幅: 36.56%

【下一步操作】
├─ Step 0: 查询 05:24 前后3小时的变更事件
├─ Step 2: 定位具体下跌阶段
└─ Step 3: 排查召回渠道异常
```

### 3. Step 2 阶段异常增强

当某个 phase 异常时，输出：

```
【🔴 recall 阶段异常详情】
├─ 当前平均:   4.31%
├─ 昨天平均:   4.51%
├─ 绝对差异:   -0.20%
├─ 相对跌幅:   5.24%
├─ 判定阈值:   >5% (SOP Step 2)
└─ 判定结果:   recall 阶段显著下跌，需进一步排查
```

### 4. Step 3 渠道异常增强

当召回渠道异常时，输出：

```
【🔴 DSSM_EMB_I2I_OUTOF24H_1000PV 渠道异常详情】
├─ 当前召回量: 1.5/min
├─ 昨天召回量: 62.0/min
├─ 衰减比率:   0.02 (阈值: <0.9)
├─ 绝对减少:   60.5/min (98.0%)
└─ 判定依据:   SOP Step 3 - 召回渠道衰减超过10%需排查
```

## 可信度提升

| 方面 | 改进前 | 改进后 |
|------|--------|--------|
| 具体数值 | ❌ 无 | ✅ 精确到小数点后2位 |
| 对比基准 | ❌ 无 | ✅ 昨天同期明确值 |
| 判定依据 | ❌ 简略 | ✅ 引用 SOP 具体条款 |
| 影响范围 | ❌ 无 | ✅ 服务/渠道/Pod |
| 可操作指令 | ❌ 无 | ✅ 下一步具体操作 |
