A0
用户 安装「XPILOT 告警诊断」SKILL
用户在 ClawHub 找到并安装 XPILOT 告警诊断 SKILL，触发 Claw 感知流程
依赖：ClawHub 上需发布「XPILOT 告警诊断」SKILL（包含告警规则拉取 + XPILOT 上传能力）
A1
Claw 主动 感知用户负责的服务 + 未绑定规则
SKILL 安装后，Claw 拉取用户负责的服务列表，查询 Xray 中未绑定排障 SKILL 的告警规则，主动告知用户
💬 Claw 说：「检测到你负责的 3 个服务 有 7 条告警规则 未绑定排障 SKILL，要帮你配置吗？」
A2
Claw 引导 选择并上传排障 SKILL
Claw 展示可用的排障 SKILL 列表（已有的 + 推荐新建的），用户选择后 Claw 自动调用 XPILOT API 完成上传
若用户已有本地 SKILL 文件，直接上传；若没有，Claw 根据服务特征推荐匹配的公共 SKILL 模板
A3
Claw 引导 选择告警规则完成绑定
Claw 拉取 Xray 告警规则列表，按服务分组展示，用户勾选规则后 Claw 调用 XPILOT API 完成绑定
Claw 可根据规则名称和 SKILL 描述智能预选——如「连接池」规则自动匹配「连接池排障 SKILL」
A4
Claw 闭环 给出完整链路预期
绑定完成后，Claw 告知用户后续会发生什么：触发告警 → 推送到哪个 Hi IM 群 → 诊断耗时预估 → 历史成功率
💬 Claw 说：「配置完成！下次 creator-service 连接池告警 触发后，排障 SKILL 将自动执行（预计 3-4 分钟），诊断结果推送到 推荐业务告警群，历史成功率 89%」
关键依赖
依赖1 ClawHub SKILL 发布
需在 ClawHub 发布「XPILOT 告警诊断」SKILL，内置：拉取告警规则、上传 SKILL 到 XPILOT、绑定规则三个能力
依赖2 XPILOT 开放 API
XPILOT 需提供：SKILL 上传 API、规则绑定 API、用户服务列表查询 API，供 Claw SKILL 调用
依赖3 Xray 告警规则查询
现有 alarm-rule-search SKILL 已支持，可直接复用。需要关联用户负责的服务范围
依赖4 用户服务归属
需要从服务树或 oncall 系统获取「用户负责哪些服务」，用于过滤告警规则范围
🔗
B / C 链路完全不变
配置完成后，后续链路与现有方案一致：告警触发 → Hi IM 群推送告警卡片 → SKILL 自动执行诊断 → 用户点卡片进侧边栏追问。
Claw 只负责 A 链路（配置阶段），不接管告警推送和诊断交互。
